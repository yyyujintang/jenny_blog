## Motivation

具身智能体需要同时具备抽象推理和具体执行的能力，就像人类可以先在脑海中规划行动序列，再根据实际环境调整。

然而，现有工作通常缺乏这种将抽象推理与具身执行相结合的基础设施。

这限制了智能体在复杂环境中的泛化能力，使其难以将学到的抽象知识应用于具体的视觉场景。

## Contribution

- **ALFWorld模拟器**: 引入一个新模拟器，允许智能体在TextWorld中学习抽象的文本策略，并在ALFRED基准的丰富视觉环境中执行目标。
- **BUTLER智能体**: 提出一种新的BUTLER智能体，其在TextWorld中学到的抽象知识可以直接对应到具体的、视觉接地的动作。
- **提升泛化能力**: 实验证明，这种方法比仅在视觉环境中训练能带来更好的智能体泛化能力。
- **模块化设计**: BUTLER的简单模块化设计将问题分解，使研究人员可以专注于改进管道的每个部分（语言理解、规划、导航和视觉场景理解）。

## Method

本文引入了ALFWorld，一个结合了TextWorld和ALFRED的模拟器，并提出了BUTLER智能体。

1.  **抽象策略学习**:
    *   智能体首先在TextWorld中学习抽象的文本策略。TextWorld提供了一个纯文本环境，智能体可以在其中进行高级规划和推理，而无需处理视觉细节。
    *   学习到的策略是基于文本的，例如“去厨房”、“拿起苹果”等。

2.  **具身环境执行**:
    *   ALFWorld将TextWorld的抽象指令映射到ALFRED的具身视觉环境。
    *   BUTLER智能体利用其在TextWorld中学到的抽象知识，在ALFRED的视觉环境中执行具体的动作，例如导航到厨房、识别苹果并拿起。

3.  **BUTLER智能体架构**:
    *   BUTLER采用模块化设计，将语言理解、规划、导航和视觉场景理解等任务分解。
    *   这种设计允许智能体将抽象的文本指令转化为一系列可在视觉环境中执行的低级动作。

## Comparison with Previous Work

- **vs. 仅在具身环境中训练**: 现有具身智能体通常直接在视觉环境中训练，缺乏抽象推理能力。ALFWorld和BUTLER通过结合TextWorld的抽象规划，显著提升了智能体的泛化能力，尤其是在未见过的场景中。
- **vs. 纯文本环境**: TextWorld等纯文本环境虽然能进行抽象推理，但无法处理真实的视觉信息和具身交互。ALFWorld弥补了这一差距，将抽象规划与具身执行无缝结合。
- **vs. ALFRED基准**: ALFRED基准提供了丰富的视觉环境和具身任务，但缺乏一个能让智能体同时进行抽象文本规划和具身执行的统一框架。ALFWorld提供了这样的基础设施。

## Motivation

Embodied agents require the ability to reason abstractly and execute concretely, much like humans can imagine action sequences and then adapt their plans to fit a specific scene.

However, existing work often lacks the necessary infrastructure to combine abstract reasoning with embodied execution.

This limitation restricts an agent's generalization capabilities in complex environments, making it difficult to apply learned abstract knowledge to specific visual scenes.

## Contribution

- **ALFWorld Simulator**: Introduction of a new simulator that enables agents to learn abstract, text-based policies in TextWorld and then execute goals from the ALFRED benchmark in a rich visual environment.
- **BUTLER Agent**: Proposal of a new BUTLER agent whose abstract knowledge, learned in TextWorld, corresponds directly to concrete, visually grounded actions.
- **Improved Generalization**: Empirical demonstration that this approach fosters better agent generalization than training only in the visually grounded environment.
- **Modular Design**: BUTLER's simple, modular design factors the problem to allow researchers to focus on models for improving every piece of the pipeline (language understanding, planning, navigation, and visual scene understanding).

## Method

This paper introduces ALFWorld, a simulator combining TextWorld and ALFRED, and proposes the BUTLER agent.

1.  **Abstract Policy Learning**:
    *   The agent first learns abstract, text-based policies in TextWorld. TextWorld provides a purely textual environment where the agent can perform high-level planning and reasoning without dealing with visual details.
    *   The learned policies are text-based, such as "go to kitchen" or "pick up apple."

2.  **Embodied Environment Execution**:
    *   ALFWorld maps the abstract instructions from TextWorld to ALFRED's embodied visual environment.
    *   The BUTLER agent utilizes its abstract knowledge learned in TextWorld to execute concrete actions in ALFRED's visual environment, such as navigating to the kitchen, identifying an apple, and picking it up.

3.  **BUTLER Agent Architecture**:
    *   BUTLER employs a modular design, decomposing tasks like language understanding, planning, navigation, and visual scene understanding.
    *   This design allows the agent to translate abstract text instructions into a sequence of low-level actions executable in a visual environment.

## Comparison with Previous Work

- **vs. Embodied-Only Training**: Existing embodied agents are typically trained directly in visual environments, lacking abstract reasoning capabilities. ALFWorld and BUTLER significantly improve generalization, especially in unseen scenarios, by integrating abstract planning from TextWorld.
- **vs. Pure Textual Environments**: Pure textual environments like TextWorld enable abstract reasoning but cannot handle real visual information and embodied interaction. ALFWorld bridges this gap by seamlessly combining abstract planning with embodied execution.
- **vs. ALFRED Benchmark**: The ALFRED benchmark offers rich visual environments and embodied tasks but lacks a unified framework for agents to perform both abstract text-based planning and embodied execution. ALFWorld provides this necessary infrastructure.
