## Motivation

大型语言模型（LLMs）正被部署到日益复杂的自主agent应用中，其中长程记忆对于实现强大性能至关重要。

然而，实际应用与当前agent记忆评估标准之间存在显著差距，现有基准主要关注以对话为中心的人机交互。

实际上，agent记忆由连续的agent-环境交互流组成，这些交互主要由机器生成的表示构成。

## Contribution

- **AMA-Bench基准**: 首次全面评估LLM在真实agentic应用中长程记忆能力的基准。
- **真实世界agentic轨迹**: 包含来自代表性agentic应用的真实世界轨迹，并配有专家策划的QA。
- **合成agentic轨迹**: 包含可扩展到任意时间范围的合成轨迹，并配有基于规则的QA。
- **AMA-Agent记忆系统**: 提出一种有效的记忆系统，其特点是包含因果图（causality graph）和工具增强检索（tool-augmented retrieval）。

## Method

### AMA-Bench基准设计
1.  **真实世界Agentic轨迹**:
    *   从Embodied AI、Gaming、Web Task Execution、Software Engineering、Open World Tool QA、Text 2 SQL等六个代表性agentic应用中收集真实轨迹。
    *   通过专家标注生成高质量的QA对，用于评估agent的记忆能力。
2.  **合成Agentic轨迹**:
    *   设计可扩展到任意时间范围的合成轨迹，以弥补真实数据在长程记忆评估上的局限性。
    *   利用规则生成QA对，确保评估的客观性和可控性。

### AMA-Agent记忆系统
1.  **因果图 (Causality Graph)**:
    *   构建一个表示agent动作、观察和状态之间因果关系的图结构。
    *   通过因果关系来组织和检索记忆，解决传统相似度检索的局限性。
2.  **工具增强检索 (Tool-Augmented Retrieval)**:
    *   结合外部工具（如知识图谱、数据库）来增强记忆检索的准确性和广度。
    *   允许agent在需要时调用工具，以获取更精确或更全面的信息，克服LLM内部知识的限制。

## Comparison with Previous Work

- **vs. 现有对话中心基准**: 现有基准如LoCoMo、LongMemEval等主要关注人机对话，而AMA-Bench专注于agent-环境交互，更符合实际agentic应用场景。
- **vs. 现有长上下文基准**: 诸如QuALITY、LongBench v2等基准侧重于长文本理解，而AMA-Bench则关注agent在连续交互流中的记忆、因果推理和状态更新能力。
- **vs. 基于相似度的检索系统**: 现有许多记忆系统依赖于相似度检索，这在处理因果关系和客观信息时存在局限性。AMA-Agent通过引入因果图和工具增强检索来克服这些问题。

## Motivation

Large Language Models (LLMs) are increasingly deployed as autonomous agents in complex applications, where long-horizon memory is critical for strong performance.

However, a significant gap exists between practical applications and current evaluation standards for agent memory. Existing benchmarks primarily focus on dialogue-centric, human-agent interactions.

In reality, agent memory consists of a continuous stream of agent-environment interactions, primarily composed of machine-generated representations.

## Contribution

-   **AMA-Bench Benchmark**: The first comprehensive benchmark to evaluate LLMs' long-horizon memory in real agentic applications.
-   **Real-world Agentic Trajectories**: Features a set of real-world agentic trajectories across representative agentic applications, paired with expert-curated QA.
-   **Synthetic Agentic Trajectories**: Includes a set of synthetic agentic trajectories that scale to arbitrary horizons, paired with rule-based QA.
-   **AMA-Agent Memory System**: Proposes an effective memory system featuring a causality graph and tool-augmented retrieval.

## Method

### AMA-Bench Benchmark Design
1.  **Real-world Agentic Trajectories**:
    *   Collected from six representative agentic applications, including Embodied AI, Gaming, Web Task Execution, Software Engineering, Open World Tool QA, and Text 2 SQL.
    *   Expert-curated QA pairs are generated to evaluate the agent's memory capabilities.
2.  **Synthetic Agentic Trajectories**:
    *   Designed synthetic trajectories that can scale to arbitrary horizons, addressing the limitations of real-world data for long-horizon memory evaluation.
    *   Rule-based QA pairs are generated to ensure objectivity and control in evaluation.

### AMA-Agent Memory System
1.  **Causality Graph**:
    *   Constructs a graph structure representing causal relationships between agent actions, observations, and states.
    *   Organizes and retrieves memories based on causality, overcoming the limitations of traditional similarity-based retrieval.
2.  **Tool-Augmented Retrieval**:
    *   Integrates external tools (e.g., knowledge graphs, databases) to enhance the accuracy and breadth of memory retrieval.
    *   Allows the agent to invoke tools when needed to obtain more precise or comprehensive information, addressing the limitations of LLM's internal knowledge.

## Comparison with Previous Work

-   **vs. Existing Dialogue-Centric Benchmarks**: Benchmarks like LoCoMo and LongMemEval primarily focus on human-agent dialogue. AMA-Bench, in contrast, focuses on agent-environment interactions, which is more aligned with real-world agentic applications.
-   **vs. Existing Long-Context Benchmarks**: Benchmarks such as QuALITY and LongBench v2 emphasize long text understanding. AMA-Bench, however, targets an agent's ability to remember, infer causality, and update states within continuous interaction streams.
-   **vs. Similarity-Based Retrieval Systems**: Many existing memory systems rely on similarity-based retrieval, which has limitations in handling causality and objective information. AMA-Agent addresses these issues by introducing a causality graph and tool-augmented retrieval.
