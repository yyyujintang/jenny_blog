## Method

### MemoryAgentBench设计

1.  **定义核心记忆能力**:
    *   **准确检索 (Accurate Retrieval, AR)**: 评估Agent从历史信息中精确提取相关知识的能力。
    *   **测试时学习 (Test-Time Learning, TTL)**: 衡量Agent在交互过程中动态学习和适应新信息的能力。
    *   **长程理解 (Long-Range Understanding, LRU)**: 考察Agent对长时间跨度内复杂上下文的理解和推理能力。
    *   **选择性遗忘 (Selective Forgetting, SF)**: 评估Agent识别并忽略过时或不相关信息，专注于最新相关信息的能力。
2.  **构建评估数据集**:
    *   为每种核心记忆能力精心挑选或构建了多个数据集，确保任务多样性和挑战性。例如，AR任务包括SH-Doc QA和MH-Doc QA；TTL任务包括BANKING77和CLINC150等。
    *   数据集设计考虑了长上下文深度和多轮交互的特点。
3.  **评估协议**:
    *   采用增量式多轮交互范式，模拟Agent在真实世界中的持续对话和任务执行过程。
    *   为每个任务定义了清晰的评估指标（如Accuracy, F1-Score, Recall@K），以量化Agent的表现。*(基于标题与领域常识推断,未读全文)*

## Motivation

Large Language Models (LLMs) show immense potential as agents in complex tasks, yet their memory capabilities remain a bottleneck. Existing evaluation benchmarks often focus on single memory dimensions or simplistic scenarios, failing to comprehensively reflect agent performance in realistic multi-turn interactions.

LLM agents need to process long contexts, understand historical information, and dynamically update their knowledge and behavior based on new information. This requires agents to possess various memory capabilities, including accurate retrieval, test-time learning, long-range understanding, and selective forgetting.

Therefore, there is a critical need for a benchmark that can systematically and multidimensionally evaluate the memory capabilities of LLM agents to drive advancements in agent memory mechanisms. *(Inferred from title/venue; full paper not read.)*

## Method

### MemoryAgentBench Design

1.  **Define Core Memory Competencies**:
    *   **Accurate Retrieval (AR)**: Evaluates an agent's ability to precisely extract relevant knowledge from historical information.
    *   **Test-Time Learning (TTL)**: Measures an agent's capacity to dynamically learn and adapt to new information during interactions.
    *   **Long-Range Understanding (LRU)**: Assesses an agent's ability to comprehend and reason over complex contexts spanning long durations.
    *   **Selective Forgetting (SF)**: Evaluates an agent's ability to identify and disregard outdated or irrelevant information, focusing on the most current and pertinent details.
2.  **Construct Evaluation Datasets**:
    *   Carefully selected or constructed multiple datasets for each core memory competency, ensuring task diversity and challenge. For example, AR tasks include SH-Doc QA and MH-Doc QA; TTL tasks include BANKING77 and CLINC150.
    *   Dataset design considered characteristics like long context depth and multi-turn interactions.
3.  **Evaluation Protocol**:
    *   Adopted an incremental multi-turn interaction paradigm to simulate an agent's continuous dialogue and task execution in real-world settings.
    *   Defined clear evaluation metrics (e.g., Accuracy, F1-Score, Recall@K) for each task to quantify agent performance. *(Inferred from title/venue; full paper not read.)*
