## Motivation

当前LLM智能体的记忆系统大多依赖于少量静态、人工设计的操作来提取记忆。这些固定的程序硬编码了人类关于存储什么以及如何修改记忆的先验知识。
这使得它们在面对多样化的交互模式时显得僵化，并且在处理长历史记录时效率低下。
因此，需要一种更灵活、更具适应性的记忆管理机制，使智能体能够根据实际需求学习和演化其记忆操作。

## Contribution

- **可学习和可演化的记忆技能**: 将传统的记忆操作重新定义为可学习和可演化的“记忆技能”，这些技能是结构化且可重用的例程，用于从交互轨迹中提取、整合和修剪信息。
- **三模块闭环架构**: 提出了一个包含Controller、Executor和Designer的闭环框架，实现了技能选择策略和技能集本身的持续改进。
- **Designer模块**: 引入了一个独特的Designer模块，它能周期性地审查记忆提取中的“疑难案例”，并通过提出改进和新技能来演化技能集。
- **广泛的实验验证**: 在LoCoMo、LongMemEval、HotpotQA和ALFWorld等多个基准测试上，MemSkill显著提升了任务性能，并展现出良好的泛化能力。

## Method

MemSkill的核心思想是将记忆操作视为可学习和可演化的技能。它包含三个主要组件：

1.  **Controller**:
    *   学习如何从现有技能集中选择一小组相关的记忆技能。
    *   根据当前任务和上下文，动态决定哪些技能最适合用于记忆提取。
2.  **Executor**:
    *   一个基于LLM的模块。
    *   根据Controller选择的技能，生成技能引导的记忆。
    *   负责实际执行记忆的提取、整合和修剪过程。
3.  **Designer**:
    *   周期性地审查Controller和Executor在记忆提取过程中遇到的“疑难案例”（即选择的技能导致不正确或不完整的记忆）。
    *   通过分析这些案例，Designer提出对现有技能的改进（refinements）或创建全新的技能。
    *   这个模块是MemSkill实现技能集自演进的关键。

MemSkill通过这三个模块形成一个闭环过程：Controller选择技能，Executor执行记忆操作，Designer根据执行结果反馈来演化技能集和改进Controller的技能选择策略，从而实现持续的自我改进。

## Comparison with Previous Work

- **vs. 静态、人工设计的记忆系统**: 传统的LLM智能体记忆系统通常依赖于一组固定的、由人类专家预设的记忆操作（如简单的RAG、摘要、遗忘策略）。MemSkill通过将这些操作转化为可学习和可演化的技能，克服了其僵化性和低效性。
- **vs. 仅学习技能选择的智能体**: 一些现有的智能体系统可能学习如何选择工具或技能，但它们的技能集本身是固定的。MemSkill的独特之处在于其Designer模块能够主动演化和扩展技能集，实现更深层次的适应性。
- **vs. 缺乏闭环反馈机制的系统**: 许多记忆管理方法缺乏一个明确的机制来根据性能反馈持续改进记忆策略或结构。MemSkill的Controller-Executor-Designer闭环设计确保了记忆管理策略和技能集的持续优化。
- **vs. 仅关注记忆内容而非操作**: 现有研究可能更多关注如何存储和检索记忆内容（如不同类型的episodic memory、semantic memory），而MemSkill则将焦点放在了管理这些记忆的“操作”本身，并使其变得智能和可演化。

## Motivation

Most current Large Language Model (LLM) agent memory systems rely on a small set of static, hand-designed operations for extracting memory. These fixed procedures hard-code human priors about what to store and how to revise memory.
This rigidity makes them inflexible under diverse interaction patterns and inefficient when dealing with long histories.
Therefore, there is a clear need for a more adaptive and flexible memory management mechanism that allows agents to learn and evolve their memory operations based on actual requirements.

## Contribution

- **Learnable and Evolvable Memory Skills**: Redefines traditional memory operations as learnable and evolvable "memory skills," which are structured and reusable routines for extracting, consolidating, and pruning information from interaction traces.
- **Three-Module Closed-Loop Architecture**: Proposes a closed-loop framework comprising a Controller, Executor, and Designer, enabling continuous improvement of both the skill selection policy and the skill set itself.
- **Designer Module**: Introduces a unique Designer module that periodically reviews "hard cases" in memory extraction and evolves the skill set by proposing refinements and new skills.
- **Extensive Experimental Validation**: MemSkill significantly improves task performance and demonstrates strong generalization across multiple benchmarks, including LoCoMo, LongMemEval, HotpotQA, and ALFWorld.

## Method

MemSkill's core idea is to treat memory operations as learnable and evolvable skills. It comprises three main components:

1.  **Controller**:
    *   Learns to select a small set of relevant memory skills from the existing skill set.
    *   Dynamically decides which skills are most appropriate for memory extraction based on the current task and context.
2.  **Executor**:
    *   An LLM-based module.
    *   Produces skill-guided memories based on the skills selected by the Controller.
    *   Responsible for the actual execution of memory extraction, consolidation, and pruning processes.
3.  **Designer**:
    *   Periodically reviews "hard cases" encountered by the Controller and Executor during memory extraction (i.e., instances where selected skills yield incorrect or incomplete memories).
    *   By analyzing these cases, the Designer proposes refinements to existing skills or creates entirely new skills.
    *   This module is crucial for MemSkill to achieve self-evolution of its skill set.

Together, MemSkill forms a closed-loop procedure through these three modules: the Controller selects skills, the Executor performs memory operations, and the Designer provides feedback based on execution results to evolve the skill set and improve the Controller's skill selection policy, thereby achieving continuous self-improvement.

## Comparison with Previous Work

- **vs. Static, Hand-Designed Memory Systems**: Traditional LLM agent memory systems typically rely on a fixed set of pre-defined memory operations (e.g., simple RAG, summarization, forgetting policies). MemSkill overcomes their rigidity and inefficiency by transforming these operations into learnable and evolvable skills.
- **vs. Agents Learning Only Skill Selection**: Some existing agent systems may learn how to select tools or skills, but their skill sets themselves are static. MemSkill's uniqueness lies in its Designer module, which actively evolves and expands the skill set, enabling deeper adaptability.
- **vs. Systems Lacking Closed-Loop Feedback Mechanisms**: Many memory management approaches lack a clear mechanism for continuously improving memory strategies or structures based on performance feedback. MemSkill's Controller-Executor-Designer closed-loop design ensures continuous optimization of both memory management strategies and the skill set.
- **vs. Focusing Solely on Memory Content over Operations**: Existing research often focuses more on how to store and retrieve memory content (e.g., different types of episodic memory, semantic memory). MemSkill, however, shifts the focus to the "operations" that manage these memories, making them intelligent and evolvable.
