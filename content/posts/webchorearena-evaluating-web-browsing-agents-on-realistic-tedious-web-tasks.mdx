## Motivation

当前的AI代理在处理复杂的、需要多步骤规划和长期记忆的网页任务时面临挑战。现有基准测试往往过于简化，未能捕捉真实世界网页交互的复杂性和繁琐性。

为了推动AI代理在真实网页环境中的能力，我们需要一个更具挑战性、更能反映用户实际操作的评估平台。这对于开发能够有效管理信息、执行复杂指令的智能代理至关重要。

## Contribution

*   **新的基准测试WebChoreArena**: 引入了一套包含逼真且繁琐的网页任务的基准测试，旨在评估AI代理在复杂网页环境中的表现。
*   **强调长期记忆与规划**: 任务设计侧重于需要代理进行多步骤规划、信息检索和长期记忆管理的能力。
*   **现实世界任务模拟**: 任务场景模拟了用户在日常生活中可能遇到的各种耗时且需要细致操作的网页任务，例如预订、比较商品、填写表格等。
*   **评估框架**: 提供了一个全面的评估框架，用于衡量代理在任务完成度、效率和错误率等方面的表现。

## Method

WebChoreArena的设计核心是模拟真实用户行为，通过以下步骤构建和评估任务：

1.  **任务收集与设计**:
    *   从真实用户行为数据中识别常见的、耗时的网页任务模式。
    *   将这些模式转化为具体的、可操作的任务指令，确保任务的复杂性和多样性。
    *   任务涵盖了信息查找、数据输入、多页面导航、决策制定等多种交互类型。
2.  **环境构建**:
    *   利用真实的网页环境作为代理的操作平台，避免使用模拟或简化环境。
    *   确保网页内容的动态性和交互性，反映真实世界的复杂性。
3.  **代理评估**:
    *   代理接收自然语言的任务指令，并尝试在网页环境中完成任务。
    *   通过记录代理的操作序列、最终状态和任务完成情况来评估其性能。
    *   引入了细粒度的评估指标，包括任务成功率、完成时间、操作步骤数和错误类型等。

## Motivation

Current AI agents struggle with complex web tasks that require multi-step planning and long-term memory. Existing benchmarks are often oversimplified, failing to capture the intricacy and tediousness of real-world web interactions.

To advance AI agents' capabilities in authentic web environments, a more challenging evaluation platform that reflects actual user operations is necessary. This is crucial for developing intelligent agents capable of effectively managing information and executing complex instructions.

## Contribution

*   **Novel Benchmark WebChoreArena**: Introduces a new benchmark comprising realistic and tedious web tasks designed to evaluate AI agents' performance in complex web environments.
*   **Emphasis on Long-Term Memory and Planning**: Task design specifically targets agents' abilities in multi-step planning, information retrieval, and long-term memory management.
*   **Real-World Task Simulation**: Task scenarios simulate various time-consuming and meticulous web tasks users might encounter daily, such as booking, comparing products, and filling out forms.
*   **Evaluation Framework**: Provides a comprehensive evaluation framework to measure agent performance in terms of task completion, efficiency, and error rates.

## Method

WebChoreArena's core design simulates real user behavior, constructing and evaluating tasks through the following steps:

1.  **Task Collection and Design**:
    *   Identifies common, time-consuming web task patterns from real user behavior data.
    *   Translates these patterns into specific, actionable task instructions, ensuring complexity and diversity.
    *   Tasks cover various interaction types, including information lookup, data entry, multi-page navigation, and decision-making.
2.  **Environment Construction**:
    *   Utilizes real web environments as the operational platform for agents, avoiding simulated or simplified settings.
    *   Ensures the dynamic and interactive nature of web content, reflecting real-world complexity.
3.  **Agent Evaluation**:
    *   Agents receive natural language task instructions and attempt to complete them within the web environment.
    *   Performance is assessed by recording agent action sequences, final states, and task completion status.
    *   Fine-grained evaluation metrics are introduced, including task success rate, completion time, number of operational steps, and error types.

*   **vs. MiniWoB++ / ALFWorld**: WebChoreArena features higher task complexity, closer to real-world web interactions rather than simplified environments or games. It focuses more on "tedious" tasks requiring long-term memory and multi-step planning, beyond simple clicks or information extraction.
*   **vs. SayCan / Gato**: Existing general-purpose agents often excel in specific domains, but their generalization capabilities and efficiency in open-ended, unstructured web environments still need improvement. WebChoreArena provides a more challenging evaluation scenario.
