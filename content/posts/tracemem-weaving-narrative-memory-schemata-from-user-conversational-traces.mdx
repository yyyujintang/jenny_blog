## Motivation

大型语言模型（LLM）在处理长期对话时面临瓶颈，因为其有限的上下文窗口难以有效管理长时间的对话历史。

现有记忆系统通常将交互视为孤立的片段，未能捕捉对话流中潜在的叙事连贯性。这导致LLM在需要深层叙事理解的任务中表现不佳。

## Contribution

- **认知启发式框架**: 提出了TraceMem，一个受认知启发的框架，通过三阶段流程从用户对话痕迹中编织结构化的叙事记忆图式。
- **三阶段记忆处理管道**:
    - **短期记忆处理**: 采用演绎式主题分割来划分事件边界并提取语义表示。
    - **突触记忆巩固**: 将事件总结为情景记忆，并将其与语义一起提炼成用户特定的痕迹。
    - **系统记忆巩固**: 利用两阶段分层聚类将这些痕迹组织成连贯的、随时间演变的叙事线索，并归纳到统一主题下。
- **结构化用户记忆卡**: 将叙事线索封装成结构化的用户记忆卡，形成叙事记忆图式。
- **Agentic搜索机制**: 提供了一种Agentic搜索机制，以增强记忆利用时的推理过程。
- **SOTA性能**: 在LoCoMo基准测试中实现了最先进的性能，特别是在多跳和时间推理方面超越了基线。

## Method

TraceMem通过一个三阶段管道来构建叙事记忆图式：

1.  **短期记忆处理 (Short-term Memory Processing)**:
    *   采用演绎式主题分割方法来划分对话中的事件边界。
    *   从每个事件中提取语义表示，作为后续记忆巩固的基础。

2.  **突触记忆巩固 (Synaptic Memory Consolidation)**:
    *   将短期记忆处理后的事件总结为情景记忆（episodic memories）。
    *   将这些情景记忆与提取出的语义一起提炼成用户特定的痕迹（user-specific traces）。

3.  **系统记忆巩固 (Systems Memory Consolidation)**:
    *   利用两阶段分层聚类（two-stage hierarchical clustering）方法。
    *   将用户特定的痕迹组织成连贯的、随时间演变的叙事线索（narrative threads），并归纳到统一的主题下。
    *   这些叙事线索最终被封装成结构化的用户记忆卡（user memory cards），形成完整的叙事记忆图式。

### 记忆利用 (Memory Utilization)

*   提供了一个**Agentic搜索机制**，用于在推理过程中检索和利用构建好的叙事记忆图式。

## Motivation

Large Language Models (LLMs) face a significant bottleneck in sustaining long-term interactions due to their limited context windows, which struggle to manage extended dialogue histories.

Existing memory systems often treat interactions as disjointed snippets, failing to capture the underlying narrative coherence of the dialogue stream. This deficiency hinders LLMs' performance in tasks requiring deep narrative comprehension.

## Contribution

-   **Cognitively-Inspired Framework**: Proposed TraceMem, a cognitively-inspired framework that weaves structured, narrative memory schemata from user conversational traces through a three-stage pipeline.
-   **Three-Stage Memory Processing Pipeline**:
    -   **Short-term Memory Processing**: Employs a deductive topic segmentation approach to demarcate episode boundaries and extract semantic representation.
    -   **Synaptic Memory Consolidation**: Summarizes episodes into episodic memories before distilling them alongside semantics into user-specific traces.
    -   **Systems Memory Consolidation**: Utilizes two-stage hierarchical clustering to organize these traces into coherent, time-evolving narrative threads under unifying themes.
-   **Structured User Memory Cards**: Encapsulates narrative threads into structured user memory cards, forming narrative memory schemata.
-   **Agentic Search Mechanism**: Provides an agentic search mechanism to enhance the reasoning process during memory utilization.
-   **State-of-the-Art Performance**: Achieves state-of-the-art performance on the LoCoMo benchmark, particularly excelling in multi-hop and temporal reasoning compared to baselines.

## Method

TraceMem constructs narrative memory schemata through a three-stage pipeline:

1.  **Short-term Memory Processing**:
    *   Employs a deductive topic segmentation approach to demarcate episode boundaries within the dialogue.
    *   Extracts semantic representations from each episode, forming the basis for subsequent memory consolidation.

2.  **Synaptic Memory Consolidation**:
    *   Summarizes the processed episodes into episodic memories.
    *   Distills these episodic memories, along with their semantic representations, into user-specific traces.

3.  **Systems Memory Consolidation**:
    *   Utilizes a two-stage hierarchical clustering method.
    *   Organizes the user-specific traces into coherent, time-evolving narrative threads under unifying themes.
    *   These narrative threads are then encapsulated into structured user memory cards, forming the complete narrative memory schemata.

### Memory Utilization

*   An **agentic search mechanism** is provided to retrieve and utilize the constructed narrative memory schemata during the reasoning process.

## Comparison with Previous Work

-   **vs. Existing Memory Systems**: Current systems often treat interactions as isolated snippets, failing to capture dialogue narrative coherence. TraceMem addresses this by building structured narrative memory schemata.
-   **vs. Traditional LLM Context Windows**: Traditional LLMs have limited context windows, making it difficult to manage long-term dialogue history. TraceMem effectively extends LLM long-term memory capabilities through an external memory system.
-   **vs. Baselines on LoCoMo Benchmark**: TraceMem significantly outperforms existing baseline methods on the LoCoMo benchmark, especially in multi-hop and temporal reasoning, demonstrating its superiority in deep narrative comprehension.
-   **vs. A-Mem, LightMem, Nemori**: TraceMem offers unique advantages in Persona Model and Agentic Search, and while consistent with LightMem and Nemori in Episode Segmentation, it achieves superior overall performance.
