## Motivation

大型语言模型（LLM）智能体在现实世界中扮演着越来越重要的角色，但它们面临一个核心限制：无法从历史交互中学习。这意味着智能体经常会丢弃宝贵的经验，重复过去的错误。

这种“健忘”极大地阻碍了智能体在持续任务中的表现和效率，限制了它们随着时间推移变得更加智能的能力。

因此，开发一种能够有效利用历史经验，并从中提炼出可泛化知识的记忆机制至关重要。

## Contribution

- **推理记忆框架 (ReasoningBank)**：提出一个新颖的记忆框架，它从智能体自我判断的成功和失败经验中提炼出可泛化的推理策略。
- **记忆检索与整合机制**：智能体在测试时能从ReasoningBank中检索相关记忆来指导交互，并将新的学习成果整合回记忆库。
- **记忆感知测试时扩展 (MaTTS)**：引入一种机制，通过扩展智能体的交互经验来加速和多样化学习过程，分配更多计算资源以生成丰富、多样的经验，从而合成更高质量的记忆。
- **记忆与扩展的协同作用**：证明了更好的记忆能指导更有效的扩展，形成强大的协同效应，促使智能体通过记忆驱动的经验扩展实现自我进化。

## Method

### ReasoningBank 记忆机制
1.  **经验收集与自我评判**：智能体在执行任务时，会记录其交互轨迹，并根据任务结果（成功或失败）进行自我评判。
2.  **推理策略蒸馏**：
    *   对于成功的经验，ReasoningBank会从中提炼出导致成功的通用推理模式或策略。
    *   对于失败的经验，ReasoningBank会分析失败的原因，并提炼出需要避免的策略或改进点。
3.  **记忆存储**：蒸馏出的推理策略以结构化的形式存储在ReasoningBank中，方便后续检索。

### 记忆感知测试时扩展 (MaTTS)
1.  **扩展交互经验**：通过分配更多的计算资源给每个任务，智能体能够生成更丰富、更多样化的交互经验。这包括探索不同的路径、尝试不同的策略等。
2.  **对比信号生成**：多样化的经验为ReasoningBank提供了丰富的对比信号，帮助它更好地辨别哪些策略是有效的，哪些是无效的。
3.  **高质量记忆合成**：利用这些对比信号，ReasoningBank能够合成更高质量、更具泛化性的推理记忆。
4.  **记忆引导的扩展**：高质量的记忆反过来又可以指导智能体在未来的扩展过程中更有效地探索，形成一个正向循环。

## Comparison with Previous Work

- **vs. 存储原始轨迹的记忆机制**：ReasoningBank不存储原始的、未经处理的交互轨迹，而是蒸馏出可泛化的推理策略。这使得记忆更紧凑、更具知识性，避免了重复学习和存储冗余信息。
- **vs. 仅存储成功任务例程的记忆机制**：ReasoningBank不仅学习成功经验，还从失败经验中吸取教训。这提供了更全面的学习信号，帮助智能体避免重复错误，并理解策略的局限性。
- **vs. 传统RAG (Retrieval-Augmented Generation)**：传统的RAG通常检索原始文本片段或事实知识，而ReasoningBank检索的是抽象的、可泛化的推理策略，直接指导智能体的决策过程，而非仅仅提供信息。
- **vs. 缺乏自我进化能力的智能体**：现有许多智能体在完成任务后会“忘记”历史，ReasoningBank通过持续的记忆更新和学习，使智能体能够随着时间推移自我进化，表现出新兴行为。

## Motivation

Large Language Model (LLM) agents are increasingly adopted in persistent real-world roles, but they face a critical limitation: their inability to learn from accumulated interaction history. This means agents frequently discard valuable insights and repeat past errors.

This "forgetfulness" severely hinders agent performance and efficiency in continuous tasks, limiting their capacity to become more intelligent over time.

Therefore, developing a memory mechanism that can effectively leverage historical experiences and distill generalizable knowledge from them is crucial.

## Contribution

- **Reasoning Memory Framework (ReasoningBank)**: Proposes a novel memory framework that distills generalizable reasoning strategies from an agent's self-judged successful and failed experiences.
- **Memory Retrieval and Integration Mechanism**: The agent can retrieve relevant memories from ReasoningBank at test time to inform its interaction and then integrate new learnings back into the memory bank.
- **Memory-aware Test-Time Scaling (MaTTS)**: Introduces a mechanism that accelerates and diversifies the learning process by scaling up the agent's interaction experience, allocating more compute to generate abundant, diverse experiences for synthesizing higher-quality memory.
- **Synergy between Memory and Scaling**: Demonstrates that better memory guides more effective scaling, establishing a powerful synergy that enables agents to self-evolve through memory-driven experience scaling.

## Method

### ReasoningBank Memory Mechanism
1.  **Experience Collection and Self-Judgment**: As the agent executes tasks, it records its interaction trajectories and performs self-judgment based on task outcomes (success or failure).
2.  **Reasoning Strategy Distillation**:
    *   For successful experiences, ReasoningBank distills generalizable reasoning patterns or strategies that led to success.
    *   For failed experiences, ReasoningBank analyzes the causes of failure and distills strategies to avoid or areas for improvement.
3.  **Memory Storage**: The distilled reasoning strategies are stored in a structured format within ReasoningBank for easy retrieval.

### Memory-aware Test-Time Scaling (MaTTS)
1.  **Scaling Interaction Experience**: By allocating more computational resources to each task, the agent can generate richer and more diverse interaction experiences. This includes exploring different paths, trying various strategies, etc.
2.  **Contrastive Signal Generation**: Diverse experiences provide ReasoningBank with rich contrastive signals, helping it better discern which strategies are effective and which are not.
3.  **High-Quality Memory Synthesis**: Utilizing these contrastive signals, ReasoningBank can synthesize higher-quality, more generalizable reasoning memories.
4.  **Memory-Guided Scaling**: High-quality memory, in turn, can guide the agent to explore more effectively in future scaling processes, forming a positive feedback loop.

## Comparison with Previous Work

- **vs. Memory mechanisms storing raw trajectories**: ReasoningBank does not store raw, unprocessed interaction trajectories but rather distills generalizable reasoning strategies. This makes the memory more compact and knowledge-rich, avoiding redundant learning and storage.
- **vs. Memory mechanisms storing only successful task routines**: ReasoningBank learns from both successful and failed experiences. This provides more comprehensive learning signals, helping agents avoid repeating errors and understand strategy limitations.
- **vs. Traditional RAG (Retrieval-Augmented Generation)**: Traditional RAG typically retrieves raw text snippets or factual knowledge, whereas ReasoningBank retrieves abstract, generalizable reasoning strategies that directly guide the agent's decision-making process, rather than merely providing information.
- **vs. Agents lacking self-evolution capabilities**: Many existing agents "forget" history after completing tasks. ReasoningBank, through continuous memory updates and learning, enables agents to self-evolve over time, exhibiting emergent behaviors.
