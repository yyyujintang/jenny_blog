## Motivation

*   当前的网页代理基准通常侧重于单一、定义明确的任务，这与真实世界中用户指令的复杂性不符。
*   在现实世界中，用户指令可能模糊不清、需要不同程度的AI辅助，并且会随着用户心智状态的变化而演变。
*   AI代理需要能够理解用户真实意图、跟踪用户心智状态、理解用户特定例程，并将任务映射到正确的GUI元素上，才能成功完成长周期网页任务。

## Contribution

*   **新基准RealWebAssist**: 引入了一个新基准，用于评估AI代理在真实场景中遵循顺序指令的能力，涉及长周期网页交互、视觉GUI grounding和理解模糊的真实世界用户指令。
*   **真实用户指令数据集**: 包含一个从真实世界人类用户收集的顺序指令数据集，每个用户指导一个网页助手在多个网站上执行一系列任务。
*   **多维度挑战**: 强调了现有SOTA模型在理解和grounding用户指令方面的不足，揭示了在长周期网页辅助中遵循真实世界用户指令的关键挑战。

## Method

*   **数据收集**: 从真实世界用户那里收集了顺序指令数据集。每个用户被要求在多个网站上执行一系列任务，并提供口头指令。
*   **任务设计**: 任务设计旨在模拟真实世界的复杂性，包括模糊指令、需要多步骤操作以及跨多个网站的交互。
*   **评估指标**: 引入了Task Success、Progress和Step Accuracy等指标来评估代理的性能。
*   **GUI Grounding**: 代理需要将用户指令中的意图ground到网页上的具体GUI元素。
*   **用户心智状态跟踪**: 代理需要理解用户指令的演变，并跟踪用户的心智状态和特定例程。

## Motivation

*   Current web-based agent benchmarks typically focus on single, well-defined tasks, which does not reflect the complexity of real-world user instructions.
*   In real-world scenarios, human instructions can be ambiguous, require varying levels of AI assistance, and may evolve over time, reflecting changes in the user's mental state.
*   To achieve successful long-horizon web-based assistance, AI agents must be able to reason about true user intent, track user mental states, understand user-specific routines, and ground intended tasks to correct GUI elements.

## Contribution

*   **Novel Benchmark RealWebAssist**: Introduces a new benchmark designed to evaluate sequential instruction-following in realistic scenarios involving long-horizon interactions with the web, visual GUI grounding, and understanding ambiguous real-world user instructions.
*   **Real-World User Instruction Dataset**: Includes a dataset of sequential instructions collected from real-world human users, where each user instructs a web-based assistant to perform a series of tasks on multiple websites.
*   **Highlighting Challenges**: Experimental results show that state-of-the-art models struggle to understand and ground user instructions, posing critical challenges in following real-world user instructions for long-horizon web assistance.

## Method

*   **Data Collection**: Sequential instruction datasets were collected from real-world human users. Each user provided verbal instructions to a web-based assistant to perform a series of tasks across multiple websites.
*   **Task Design**: Tasks were designed to mimic real-world complexity, including ambiguous instructions, multi-step operations, and interactions across various websites.
*   **Evaluation Metrics**: Metrics such as Task Success, Progress, and Step Accuracy are introduced to evaluate agent performance.
*   **GUI Grounding**: Agents are required to ground the intent behind user instructions to specific GUI elements on the web page.
*   **User Mental State Tracking**: Agents need to understand the evolving nature of user instructions and track the user's mental state and specific routines.

## Comparison with Previous Work

*   **vs. Existing Web Agent Benchmarks**: RealWebAssist differentiates itself from existing benchmarks (e.g., ScreenSpot, WebArena, Mind2Web, WebLINX) by incorporating real users, sequential instructions, real websites, GUI grounding, and speech input.
*   **vs. Single-Task Benchmarks**: Unlike benchmarks focused on single, clearly defined tasks, RealWebAssist addresses long-horizon, ambiguous, and evolving user instructions.
*   **vs. Simulated Users**: RealWebAssist's dataset is collected from real human users, offering a more realistic evaluation compared to benchmarks relying on simulated users.
