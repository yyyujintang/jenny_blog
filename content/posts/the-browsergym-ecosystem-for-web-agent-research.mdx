## Motivation

现有Web代理基准测试存在碎片化和评估方法不一致的问题，这使得可靠的比较和可复现的结果难以实现。

特别是在利用自动化和LLM的Web代理领域，对高效评估和基准测试的需求日益增长。

Drouin et al. (2024) 提出的BrowserGym旨在通过提供统一的gym-like环境来解决这些问题。

## Contribution

- **统一的Web代理评估生态系统**: 扩展了BrowserGym，提供了一个统一的、gym-like环境，具有明确的观察和动作空间，促进跨不同基准的标准化评估。
- **AgentLab框架**: 引入了一个互补的框架AgentLab，旨在辅助Web代理的创建、测试和分析，简化开发流程。
- **基准统一与整合**: 整合了现有文献中的多个Web代理基准，确保了评估的一致性和全面的实验管理。
- **大规模多基准实验**: 首次进行了大规模、多基准的Web代理实验，比较了6个SOTA LLM在6个流行基准上的性能。
- **性能差距揭示**: 发现OpenAI和Anthropic最新模型之间存在显著性能差异，Claude-3.5-Sonnet在大多数基准上表现领先，而GPT-4o在视觉相关任务上更优。

## Method

### BrowserGym核心环境
1.  **统一接口**: BrowserGym提供了一个统一的gym-like接口，用于与Web环境交互，定义了标准的observation和action spaces。
2.  **Web环境抽象**: 将复杂的Web环境抽象为可控的RL环境，允许代理通过结构化的动作与页面元素交互。

### AgentLab框架
1.  **代理创建**: AgentLab提供工具和API，简化Web代理的开发和配置。
2.  **测试与调试**: 支持代理在BrowserGym环境中的测试，提供调试工具和日志记录功能。
3.  **分析与可视化**: 帮助研究人员分析代理行为和性能，提供可视化工具以理解代理决策。

### 基准整合
1.  **现有基准统一**: 将MiniWoB(++), WebArena, VisualWebArena, WorkArena (L1, L2, L3), WebLINX, Assistant Bench等多个流行Web代理基准整合到BrowserGym生态系统中。
2.  **标准化评估**: 通过统一的接口和评估指标，确保跨不同基准的评估一致性和可比性。

## Motivation

Existing web agent benchmarks suffer from fragmentation and inconsistent evaluation methodologies, making it challenging to achieve reliable comparisons and reproducible results.

There is a growing need for efficient evaluation and benchmarking of web agents, particularly those leveraging automation and Large Language Models (LLMs).

An earlier work by Drouin et al. (2024) introduced BrowserGym, aiming to solve these issues by providing a unified, gym-like environment.

## Contribution

- **Unified Web Agent Evaluation Ecosystem**: Extends BrowserGym to provide a unified, gym-like environment with well-defined observation and action spaces, facilitating standardized evaluation across diverse benchmarks.
- **AgentLab Framework**: Introduces AgentLab, a complementary framework that aids in agent creation, testing, and analysis, simplifying the development process.
- **Benchmark Unification and Integration**: Unifies existing web agent benchmarks from the literature, ensuring consistent evaluation and comprehensive experiment management.
- **Large-Scale Multi-Benchmark Experiment**: Conducts the first large-scale, multi-benchmark web agent experiment, comparing the performance of 6 state-of-the-art LLMs across 6 popular web agent benchmarks.
- **Discrepancy in Model Performance**: Highlights a large discrepancy between OpenAI and Anthropic's latest models, with Claude-3.5-Sonnet leading on almost all benchmarks, except on vision-related tasks where GPT-4o is superior.

## Method

### BrowserGym Core Environment
1.  **Unified Interface**: BrowserGym provides a unified, gym-like interface for interacting with web environments, defining standard observation and action spaces.
2.  **Web Environment Abstraction**: It abstracts complex web environments into controllable RL environments, allowing agents to interact with page elements through structured actions.

### AgentLab Framework
1.  **Agent Creation**: AgentLab offers tools and APIs to streamline the development and configuration of web agents.
2.  **Testing and Debugging**: Supports agent testing within the BrowserGym environment, providing debugging tools and logging capabilities.
3.  **Analysis and Visualization**: Helps researchers analyze agent behavior and performance, offering visualization tools to understand agent decisions.

### Benchmark Integration
1.  **Unification of Existing Benchmarks**: Integrates several popular web agent benchmarks, including MiniWoB(++), WebArena, VisualWebArena, WorkArena (L1, L2, L3), WebLINX, and Assistant Bench, into the BrowserGym ecosystem.
2.  **Standardized Evaluation**: Ensures consistent and comparable evaluation across different benchmarks through unified interfaces and metrics.

## Comparison with Previous Work

- **vs. Fragmented Existing Benchmarks**: Current benchmarks often lack unified evaluation methodologies and interfaces, making results difficult to compare. BrowserGym addresses this by providing a unified gym-like environment with standardized observation/action spaces.
- **vs. Single Benchmark Evaluation**: Many studies evaluate web agents on only one or a few benchmarks. This paper provides a more comprehensive and reliable performance comparison by integrating multiple popular benchmarks and conducting large-scale experiments.
- **vs. Drouin et al. (2024) BrowserGym**: This work extends Drouin et al. (2024) by not only providing the unified BrowserGym environment but also introducing the AgentLab framework, further assisting in agent creation, testing, and analysis, forming a more complete ecosystem.
- **vs. Lack of Agent Development Tools**: Existing web agent research often focuses on the models themselves, lacking systematic frameworks to aid in agent creation, testing, and analysis. AgentLab fills this gap.
