## Motivation

大型语言模型（LLMs）因其固定的上下文窗口和内存管理不足，在AI代理的交互体验中面临长期记忆能力和个性化方面的严重挑战。

现有的方法难以有效整合不同时间尺度的记忆，也无法实现记忆的动态更新和高效检索。

这些限制导致AI代理在长时间对话中缺乏上下文连贯性，并且无法根据用户历史进行个性化学习。

## Contribution

*   **分层存储架构**: 设计了短时记忆、中时记忆和长时个人记忆三层存储单元，实现了记忆的层次化整合。
*   **动态更新机制**: 提出了对话链（dialogue-chain）FIFO原则用于短时到中时记忆的更新，以及分段页面组织策略（segmented page organization strategy）用于中时到长时记忆的更新。
*   **MemoryOS框架**: 创新性地提出了一个完整的Memory Operating System，包含记忆存储、更新、检索和生成四个核心模块，实现全面高效的记忆管理。
*   **显著性能提升**: 在LoCoMo基准测试上，相对于GPT-4o-mini基线，F1平均提升49.11%，BLEU-1平均提升46.18%，证明了在长对话中上下文连贯性和个性化记忆保留的有效性。

## Method

MemoryOS通过以下四个核心模块实现全面的记忆管理：

1.  **Memory Storage (记忆存储)**:
    *   **短时记忆 (Short-term Memory, STM)**: 存储当前对话的即时信息。
    *   **中时记忆 (Mid-term Memory, MTM)**: 存储对话链，通过对话链的FIFO原则从STM更新。
    *   **长时个人记忆 (Long-term Personal Memory, LPM)**: 存储高度抽象和个性化的知识，通过分段页面组织策略从MTM更新。

2.  **Memory Updating (记忆更新)**:
    *   **STM到MTM更新**: 遵循对话链（dialogue-chain）FIFO原则，确保最新和最相关的对话片段被保留。
    *   **MTM到LPM更新**: 采用分段页面组织策略，将中时记忆中的重要信息抽象并持久化到长时记忆中。

3.  **Memory Retrieval (记忆检索)**:
    *   根据当前查询，从STM、MTM和LPM中高效检索相关信息。
    *   检索策略考虑了记忆的时间性、重要性和相关性，以提供最准确的上下文。

4.  **Memory Generation (记忆生成)**:
    *   利用检索到的多层记忆信息，结合LLM生成连贯、个性化且上下文感知的响应。
    *   确保生成的内容不仅准确，而且能体现AI代理的长期学习和个性化特征。

## Comparison with Previous Work

*   **vs. 固定上下文窗口LLMs**: 传统LLMs受限于固定上下文窗口，无法有效处理长对话和长期记忆。MemoryOS通过分层存储和动态更新机制，突破了这一限制。
*   **vs. RAG (Retrieval-Augmented Generation)**: RAG通常依赖单一的外部知识库检索，缺乏对记忆层次和动态更新的精细管理。MemoryOS提供了更复杂的记忆管理系统，能够区分不同时间尺度的记忆。
*   **vs. 现有记忆管理框架 (如MemGPT, A-Mem)**: 现有框架可能在某些方面有所侧重，但MemoryOS通过引入操作系统级的记忆管理理念，提供了更全面、更高效的分层存储和动态更新策略，尤其是在个性化记忆方面表现更优。

## Motivation

Large Language Models (LLMs) face a critical challenge due to fixed context windows and inadequate memory management. This leads to a severe shortage of long-term memory capabilities and limited personalization in AI agent interactive experiences.

Existing methods struggle to effectively integrate memories across different timescales and lack mechanisms for dynamic memory updates and efficient retrieval.

These limitations result in AI agents lacking contextual coherence in long conversations and being unable to learn personalized information based on user history.

## Contribution

*   **Hierarchical Storage Architecture**: Designed a three-level storage architecture comprising short-term, mid-term, and long-term personal memory units for hierarchical memory integration.
*   **Dynamic Updating Mechanism**: Proposed a dialogue-chain-based FIFO principle for short-term to mid-term memory updates, and a segmented page organization strategy for mid-term to long-term memory updates.
*   **MemoryOS Framework**: Innovatively proposed a complete Memory Operating System, including four key modules: Memory Storage, Updating, Retrieval, and Generation, achieving comprehensive and efficient memory management.
*   **Significant Performance Improvement**: Achieved an average improvement of 49.11% on F1 and 46.18% on BLEU-1 over GPT-4o-mini baselines on the LoCoMo benchmark, demonstrating effectiveness in contextual coherence and personalized memory retention in long conversations.

## Method

MemoryOS achieves comprehensive memory management through its four core modules:

1.  **Memory Storage**:
    *   **Short-term Memory (STM)**: Stores immediate information from the current dialogue.
    *   **Mid-term Memory (MTM)**: Stores dialogue chains, updated from STM following a dialogue-chain-based FIFO principle.
    *   **Long-term Personal Memory (LPM)**: Stores highly abstracted and personalized knowledge, updated from MTM using a segmented page organization strategy.

2.  **Memory Updating**:
    *   **STM to MTM Updates**: Follows a dialogue-chain-based FIFO principle, ensuring the most recent and relevant dialogue segments are retained.
    *   **MTM to LPM Updates**: Employs a segmented page organization strategy to abstract and persist important information from mid-term memory into long-term memory.

3.  **Memory Retrieval**:
    *   Efficiently retrieves relevant information from STM, MTM, and LPM based on the current query.
    *   Retrieval strategies consider the temporality, importance, and relevance of memories to provide the most accurate context.

4.  **Memory Generation**:
    *   Utilizes the retrieved multi-level memory information, combined with the LLM, to generate coherent, personalized, and context-aware responses.
    *   Ensures that generated content is not only accurate but also reflects the AI agent's long-term learning and personalized characteristics.

## Comparison with Previous Work

*   **vs. Fixed Context Window LLMs**: Traditional LLMs are limited by fixed context windows, making them ineffective for long conversations and long-term memory. MemoryOS overcomes this limitation through hierarchical storage and dynamic updating mechanisms.
*   **vs. RAG (Retrieval-Augmented Generation)**: RAG typically relies on a single external knowledge base for retrieval, lacking sophisticated management of memory hierarchies and dynamic updates. MemoryOS provides a more complex memory management system that differentiates between memories of different timescales.
*   **vs. Existing Memory Management Frameworks (e.g., MemGPT, A-Mem)**: While existing frameworks may focus on specific aspects, MemoryOS introduces an operating system-level memory management paradigm, offering a more comprehensive and efficient hierarchical storage and dynamic updating strategy, particularly excelling in personalized memory.
