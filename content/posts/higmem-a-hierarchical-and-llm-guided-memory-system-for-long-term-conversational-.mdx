## Motivation

长程对话LLM代理需要强大的记忆系统，以便从历史互动中恢复相关证据，同时避免无关上下文的干扰。

现有的记忆系统，即使是分层的，也主要依赖向量相似性进行检索。这常常导致检索到的证据集过于庞大。

这种庞大的证据集虽然可能增加召回率，但会降低检索精度，提高答案生成阶段的上下文成本，并使记忆难以检查和管理。

## Contribution

- **分层事件-回合记忆系统 (Two-level event-turn memory system)**: 引入两级记忆结构，将对话组织为事件和回合，提供更精细的粒度。
- **LLM引导的语义锚点 (LLM-guided semantic anchors)**: 允许LLM使用事件摘要作为语义锚点，预测哪些相关回合值得阅读，从而实现更智能的检索。
- **推理驱动的证据集 (Reasoning-driven evidence set)**: 通过LLM的推理能力，从高层事件摘要聚焦到更小、更相关的回合集，提供简洁可靠的证据。
- **高效的检索机制 (Efficient retrieval mechanism)**: 避免了纯向量检索带来的过高开销，同时显著减少了检索的回合数量。

## Method

HiGMem的核心是一个两级事件-回合记忆系统，并由LLM引导检索过程。

1.  **记忆构建 (Memory Construction)**:
    *   将历史对话组织成“事件”（events）和“回合”（turns）。
    *   每个事件包含一个摘要，作为其高层语义表示。
2.  **事件级检索 (Event-level Retrieval)**:
    *   LLM首先检查高层事件摘要，利用其语义理解能力。
    *   基于当前对话上下文和事件摘要，LLM预测哪些事件是相关的。
3.  **回合级聚焦 (Turn-level Focusing)**:
    *   在选定的相关事件内部，LLM进一步预测哪些具体的回合（turns）可能包含有用的信息。
    *   这一步避免了对所有回合进行盲目检索，显著减少了需要处理的数据量。
4.  **证据集生成 (Evidence Set Generation)**:
    *   最终，模型生成一个简洁且高度相关的证据集，用于LLM的答案生成阶段。
    *   这个过程通过推理而非纯粹的向量相似性来筛选信息，提高了检索的精度和可解释性。

## Comparison with Previous Work

- **vs. 纯向量相似性检索 (Pure Vector Similarity Retrieval)**:
    *   HiGMem通过LLM引导的推理，避免了纯向量检索常导致的“臃肿”证据集问题，显著提高了检索精度并降低了上下文成本。
- **vs. 现有分层记忆系统 (Existing Hierarchical Memory Systems)**:
    *   虽然其他系统也可能采用分层结构，但HiGMem的独特之处在于其LLM引导的语义锚点机制，能够更智能地预测相关性，而非仅仅依赖向量相似性在不同层级间跳转。
- **vs. A-Mem**:
    *   在LoCoMo10基准测试上，HiGMem在减少检索回合数量一个数量级的同时，将对抗性F1分数从0.54提升到0.78，显示出其在效率和效果上的显著优势。

## Motivation

Long-term conversational LLM agents necessitate robust memory systems to retrieve relevant evidence from historical interactions without overwhelming the answer stage with irrelevant context.

Existing memory systems, even hierarchical ones, frequently rely solely on vector similarity for retrieval. This often leads to bloated evidence sets.

Such bloated sets, while potentially increasing recall, degrade retrieval precision, escalate context costs for the answer stage, and make retrieved memories difficult to inspect and manage.

## Contribution

-   **Two-level event-turn memory system**: Introduces a hierarchical memory structure organizing dialogue into events and turns, offering finer granularity.
-   **LLM-guided semantic anchors**: Enables LLMs to leverage event summaries as semantic anchors, predicting which related turns are worth reading for smarter retrieval.
-   **Reasoning-driven evidence set**: Utilizes the LLM's reasoning capabilities to focus from high-level event summaries to a smaller, more relevant set of turns, providing concise and reliable evidence.
-   **Efficient retrieval mechanism**: Avoids the excessive overhead associated with pure vector retrieval while significantly reducing the number of retrieved turns.

## Method

HiGMem's core is a two-level event-turn memory system, with the retrieval process guided by an LLM.

1.  **Memory Construction**:
    *   Historical conversations are organized into "events" and "turns."
    *   Each event includes a summary serving as its high-level semantic representation.
2.  **Event-level Retrieval**:
    *   The LLM first inspects high-level event summaries, leveraging its semantic understanding.
    *   Based on the current conversational context and event summaries, the LLM predicts which events are relevant.
3.  **Turn-level Focusing**:
    *   Within the selected relevant events, the LLM further predicts which specific turns are likely to contain useful information.
    *   This step avoids blind retrieval across all turns, significantly reducing the amount of data to process.
4.  **Evidence Set Generation**:
    *   Finally, the model generates a concise and highly relevant evidence set for the LLM's answer generation stage.
    *   This process screens information through reasoning rather than pure vector similarity, improving retrieval precision and interpretability.

## Comparison with Previous Work

-   **vs. Pure Vector Similarity Retrieval**:
    *   HiGMem, through LLM-guided reasoning, addresses the "bloated" evidence set problem common in pure vector retrieval, significantly improving precision and reducing context costs.
-   **vs. Existing Hierarchical Memory Systems**:
    *   While other systems may also employ hierarchical structures, HiGMem's distinctiveness lies in its LLM-guided semantic anchor mechanism, which intelligently predicts relevance rather than merely relying on vector similarity for inter-level navigation.
-   **vs. A-Mem**:
    *   On the LoCoMo10 benchmark, HiGMem improves adversarial F1 from 0.54 to 0.78 while retrieving an order of magnitude fewer turns, demonstrating its significant advantages in both efficiency and effectiveness.
