## Motivation

尽管基于语言模型的代理在解决如网页导航等实际任务方面潜力巨大，但它们在处理具有复杂动作轨迹的长时任务时仍然面临挑战。

人类可以灵活地从过往经验中学习可复用的任务工作流，并利用这些工作流来指导未来的行动，从而有效地解决复杂任务。

为了让AI代理也能从这一过程中受益，我们旨在开发一种方法，使其能够像人类一样学习和应用这些可复用的任务工作流。

## Contribution

- **Agent Workflow Memory (AWM)**：提出一种新方法，通过诱导和选择性地提供常用工作流来指导代理的后续生成，从而解决长时任务。
- **灵活的部署场景**：AWM既可应用于离线场景（预先从训练示例中诱导工作流），也可应用于在线场景（根据测试查询即时诱导工作流）。
- **显著的性能提升**：在Mind2Web和WebArena两大网页导航基准测试中，AWM相对成功率分别提升了24.6%和51.1%，并减少了解决WebArena任务所需的步骤数。
- **强大的泛化能力**：在线AWM在跨任务、跨网站和跨领域评估中表现出强大的泛化能力，在训练-测试任务分布差距扩大时，绝对分数超越基线8.9至14.0个百分点。

## Method

AWM旨在通过学习和复用任务工作流来提升AI代理在长时任务中的表现。

### 工作流诱导 (Workflow Induction)

1.  **从经验中学习**：AWM从过去的任务经验中识别并诱导常用的例程，即工作流。这些工作流可以是预先从训练数据中学习的（离线），也可以是根据当前任务实时学习的（在线）。
2.  **工作流表示**：诱导出的工作流被表示为一系列动作或子任务，它们代表了解决特定类型问题的常见模式。

### 工作流检索与应用 (Workflow Retrieval and Application)

1.  **情境感知检索**：当代理遇到新任务时，AWM会根据当前任务的上下文和目标，选择性地检索最相关的工作流。
2.  **指导生成**：检索到的工作流被用作指导信息，注入到语言模型代理的生成过程中，帮助代理规划更有效、更符合逻辑的动作序列。这有助于代理避免重复错误，并更快地收敛到解决方案。

## Comparison with Previous Work

- **vs. 缺乏记忆的语言模型代理**：传统语言模型代理在处理长时任务时，往往难以维持一致性和效率，容易陷入循环或偏离目标。AWM通过引入结构化的工作流记忆，提供了更高级别的指导和规划能力。
- **vs. 仅依赖短期记忆或RAG**：虽然RAG等方法可以提供相关信息，但它们通常侧重于片段式或事实性检索。AWM则专注于学习和复用任务级别的“操作流程”，这是一种更高级别的抽象和指导，超越了简单的信息检索。
- **vs. 人工设计工作流**：一些方法依赖于人类工程师手动设计任务工作流。AWM能够自动从数据中学习这些工作流，大大减少了人工干预的需求，并能适应更广泛的任务和领域。
- **vs. 传统的RL方法**：传统的RL方法在复杂、稀疏奖励的任务中学习效率低下，且泛化能力有限。AWM通过引入可复用的工作流，为代理提供了更强的结构化先验知识，加速了学习过程并提升了泛化能力。

## Motivation

Despite the potential of language model-based agents to solve real-world tasks such as web navigation, current methods still struggle with long-horizon tasks that involve complex action trajectories.

In contrast, humans can flexibly solve complex tasks by learning reusable task workflows from past experiences and effectively using them to guide future actions.

To build agents that can similarly benefit from this process, we aim to develop a method that allows AI agents to learn and apply these reusable task workflows, much like humans do.

## Contribution

- **Agent Workflow Memory (AWM)**: We introduce a novel method that induces commonly reused routines, or workflows, and selectively provides them to the agent to guide subsequent generations, addressing long-horizon tasks.
- **Flexible Application Scenarios**: AWM is versatile, applying to both offline scenarios where workflows are induced from training examples beforehand, and online scenarios where workflows are induced from test queries on the fly.
- **Substantial Performance Improvement**: AWM significantly improves baseline results by 24.6% and 51.1% relative success rate on Mind2Web and WebArena, respectively, while also reducing the number of steps taken to solve WebArena tasks successfully.
- **Robust Generalization**: Online AWM demonstrates robust generalization in cross-task, cross-website, and cross-domain evaluations, surpassing baselines by 8.9 to 14.0 absolute points as train-test task distribution gaps widen.

## Method

AWM aims to enhance AI agent performance in long-horizon tasks by learning and reusing task workflows.

### Workflow Induction

1.  **Learning from Experience**: AWM identifies and induces commonly reused routines, or workflows, from past task experiences. These workflows can be learned beforehand from training data (offline) or in real-time based on the current task (online).
2.  **Workflow Representation**: The induced workflows are represented as sequences of actions or sub-tasks, embodying common patterns for solving specific types of problems.

### Workflow Retrieval and Application

1.  **Context-Aware Retrieval**: When an agent encounters a new task, AWM selectively retrieves the most relevant workflows based on the current task's context and objectives.
2.  **Guiding Generation**: The retrieved workflows are then used as guiding information, injected into the language model agent's generation process. This helps the agent plan more effective and logical action sequences, avoiding repetitive errors and converging to solutions more quickly.

## Comparison with Previous Work

- **vs. Memory-less Language Model Agents**: Traditional language model agents often struggle with consistency and efficiency in long-horizon tasks, frequently falling into loops or deviating from goals. AWM introduces structured workflow memory, providing a higher level of guidance and planning capability.
- **vs. Relying Solely on Short-Term Memory or RAG**: While methods like RAG can provide relevant information, they typically focus on episodic or factual retrieval. AWM, in contrast, focuses on learning and reusing task-level "operational procedures," which is a higher-level abstraction and guidance beyond simple information retrieval.
- **vs. Human-Engineered Workflows**: Some approaches rely on human engineers to manually design task workflows. AWM automates the learning of these workflows from data, significantly reducing the need for manual intervention and adapting to a broader range of tasks and domains.
- **vs. Traditional RL Methods**: Traditional RL methods often suffer from low learning efficiency and limited generalization in complex tasks with sparse rewards. By introducing reusable workflows, AWM provides agents with stronger structured prior knowledge, accelerating the learning process and improving generalization.
