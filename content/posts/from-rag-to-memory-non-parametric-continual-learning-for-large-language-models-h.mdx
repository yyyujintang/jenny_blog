## Motivation

大型语言模型 (LLMs) 持续学习和利用知识的能力是实现其全部潜力的关键。然而，LLMs 在持续学习方面面临挑战，导致 RAG 成为引入新信息的主流方式。

传统的 RAG 依赖向量检索，难以模仿人类长期记忆的动态和互联特性，特别是在理解和联想方面存在不足。

虽然一些 RAG 方法通过知识图谱等结构增强了向量嵌入，以解决理解和联想问题，但它们在基本事实记忆任务上的表现却显著下降。

## Contribution

- **全面提升记忆能力**: HippoRAG 2 在事实、理解和联想记忆任务上全面超越了标准 RAG，解决了现有结构增强 RAG 方法在事实记忆任务上性能下降的问题。
- **增强的 Personalized PageRank 算法**: 在 HippoRAG 的基础上，通过更深度的 passage 集成和更有效的 LLM 在线使用，进一步优化了 Personalized PageRank 算法。
- **接近人类长期记忆的有效性**: 将 RAG 系统推向更接近人类长期记忆的有效性，在联想记忆任务上比最先进的嵌入模型提高了 7%。
- **为非参数持续学习铺平道路**: 提出了一个框架，为 LLMs 的非参数持续学习奠定了基础。

## Method

HippoRAG 2 框架通过以下关键组件和步骤实现其增强的记忆能力：

1.  **知识图谱构建**:
    *   利用 LLM 从文本 passage 中提取三元组（triple），形成初步的知识图谱。
    *   通过 LLM 进行在线过滤，去除低质量或冗余的三元组，提高图谱的质量和效率。
2.  **图谱集成与 Passage 节点**:
    *   将原始文本 passage 作为节点集成到知识图谱中，形成一个包含实体、关系和原始文本的混合图谱。
    *   这种深度 passage 集成使得图谱能够同时捕捉结构化知识和原始文本的丰富语境信息。
3.  **增强的 Personalized PageRank 检索**:
    *   基于查询，在构建的混合图谱上运行 Personalized PageRank 算法。
    *   该算法不仅考虑实体之间的连接，还考虑 passage 节点与查询的相关性，从而实现更精准和联想性的检索。
    *   通过调整 passage 节点的权重因子，优化检索结果，使其在不同类型的记忆任务中表现更佳。
4.  **LLM 在线使用**:
    *   在图谱构建和检索过程中，LLM 被有效利用进行三元组过滤和查询理解，确保了高质量的图谱和相关性高的检索结果。
    *   检索到的信息随后被 LLM 用于生成最终答案，从而实现端到端的记忆和推理。

## Motivation

The ability to continuously acquire, organize, and leverage knowledge is a key feature of human intelligence that AI systems, particularly Large Language Models (LLMs), must approximate to unlock their full potential. However, LLMs face significant challenges in continual learning.

Retrieval-Augmented Generation (RAG) has emerged as the dominant way to introduce new information to LLMs, but its reliance on vector retrieval inherently limits its capacity to mimic the dynamic and interconnected nature of human long-term memory, especially concerning sense-making and associativity.

While recent RAG approaches augment vector embeddings with various structures like knowledge graphs to address these gaps, they often suffer from a considerable drop in performance on more basic factual memory tasks.

## Contribution

- **Comprehensive Memory Enhancement**: HippoRAG 2 comprehensively outperforms standard RAG on factual, sense-making, and associative memory tasks, addressing the performance deterioration seen in other structure-augmented RAG methods on factual memory.
- **Enhanced Personalized PageRank Algorithm**: It builds upon the Personalized PageRank algorithm from HippoRAG, enhancing it with deeper passage integration and more effective online use of an LLM.
- **Approaching Human Long-Term Memory Effectiveness**: The framework pushes RAG systems closer to the effectiveness of human long-term memory, achieving a 7% improvement in associative memory tasks over state-of-the-art embedding models.
- **Paving the Way for Non-Parametric Continual Learning**: This work lays the groundwork for non-parametric continual learning for LLMs, offering a robust solution for dynamic knowledge acquisition and utilization.

## Method

HippoRAG 2 achieves its enhanced memory capabilities through the following key components and steps:

1.  **Knowledge Graph Construction**:
    *   An LLM is used to extract triples (subject-predicate-object) from text passages, forming an initial knowledge graph.
    *   An online filtering mechanism, also powered by an LLM, prunes low-quality or redundant triples, enhancing the graph's quality and efficiency.
2.  **Graph Integration with Passage Nodes**:
    *   Original text passages are integrated as nodes within the knowledge graph, creating a hybrid graph that encompasses both structured entities/relations and the rich context of raw text.
    *   This deep passage integration allows the system to leverage both explicit knowledge and contextual information.
3.  **Enhanced Personalized PageRank Retrieval**:
    *   Given a query, a Personalized PageRank algorithm is executed on the constructed hybrid graph.
    *   This algorithm considers not only the connections between entities but also the relevance of passage nodes to the query, enabling more precise and associative retrieval.
    *   Weight factors for passage nodes are tuned to optimize retrieval results across different types of memory tasks.
4.  **Effective Online LLM Usage**:
    *   The LLM is effectively utilized throughout the graph construction and retrieval process for tasks such as triple filtering and query understanding, ensuring high-quality graph structures and relevant retrieval outcomes.
    *   The retrieved information is then fed back to the LLM for final answer generation, facilitating an end-to-end memory and reasoning pipeline.

## Comparison with Previous Work

- **vs. Standard RAG**:
    *   Standard RAG relies heavily on vector retrieval, which often falls short in sense-making and associative memory tasks. HippoRAG 2, by incorporating structured graphs and Personalized PageRank, comprehensively enhances these capabilities.
    *   HippoRAG 2 outperforms standard RAG across factual, sense-making, and associative memory tasks.
- **vs. Structure-Augmented RAG (e.g., RAPTOR, GraphRAG)**:
    *   While these methods attempt to enhance RAG with structures like knowledge graphs, they frequently lead to a significant drop in performance on basic factual memory tasks.
    *   HippoRAG 2 addresses this "unintended deterioration," maintaining or even improving factual memory while boosting sense-making and associative abilities.
- **vs. HippoRAG v1**:
    *   HippoRAG 2 builds upon the Personalized PageRank algorithm of HippoRAG v1 by introducing deeper passage integration and more effective online LLM usage.
    *   These enhancements lead to superior performance across all memory tasks, particularly demonstrating significant gains in associative memory.
