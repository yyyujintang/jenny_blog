## Motivation

当前LLM驱动的智能体记忆系统，常将记忆视为外部层，从对话中提取片段并存储，再检索到prompt中。

然而，这种方法模糊了证据与推断的界限，难以长期组织信息，并对智能体的解释能力支持有限。

为了使智能体能够积累经验、跨会话适应并超越单次问答，需要更强大的记忆机制。

## Contribution

- **结构化记忆架构**: 将记忆组织成世界事实、智能体经验、合成实体摘要和演化信念四个逻辑网络。
- **核心操作**: 定义了保留 (retain)、回忆 (recall) 和反思 (reflect) 三个核心操作，管理信息的添加、访问和更新。
- **可追溯的推理**: 通过反思层对记忆库进行推理，生成答案并以可追溯的方式更新信息。
- **性能提升**: 在LongMemEval和LoCoMo等长程对话记忆基准测试中，显著超越现有记忆架构和GPT-4o等模型。

## Method

Hindsight将智能体记忆视为一个结构化的、一流的推理基质，其核心在于四个逻辑网络和三个核心操作。

### 记忆网络
1.  **世界事实 (World Facts)**: 存储关于外部世界的客观信息。
2.  **智能体经验 (Agent Experiences)**: 记录智能体的具体交互和行动。
3.  **合成实体摘要 (Synthesized Entity Summaries)**: 对实体信息进行综合和提炼。
4.  **演化信念 (Evolving Beliefs)**: 智能体对世界的动态理解和假设。

### 核心操作
1.  **保留 (Retain)**:
    *   将对话流中的信息增量地转化为结构化、可查询的记忆库。
    *   利用时间感知和实体感知的记忆层来组织新信息。
2.  **回忆 (Recall)**:
    *   根据查询需求，从记忆库中高效地检索相关信息。
    *   支持对不同记忆网络中的信息进行访问。
3.  **反思 (Reflect)**:
    *   推理记忆库中的信息，以生成答案或更新现有信念。
    *   确保信息更新过程可追溯，增强智能体的解释能力。

## Motivation

Current LLM-based agent memory systems often treat memory as an external layer, extracting salient snippets from conversations and storing them for retrieval into the prompt.

However, this approach blurs the line between evidence and inference, struggles to organize information over long horizons, and offers limited support for agents that must explain their reasoning.

To enable agents to accumulate experience, adapt across sessions, and move beyond single-shot question answering, a more robust memory mechanism is required.

## Contribution

- **Structured Memory Architecture**: Organizes memory into four logical networks: world facts, agent experiences, synthesized entity summaries, and evolving beliefs.
- **Core Operations**: Defines three core operations—retain, recall, and reflect—that govern how information is added, accessed, and updated.
- **Traceable Reasoning**: Employs a reflection layer to reason over the memory bank, producing answers and updating information in a traceable way.
- **Performance Improvement**: Achieves significant performance gains over existing memory architectures and models like GPT-4o on long-horizon conversational memory benchmarks such as LongMemEval and LoCoMo.

## Method

Hindsight treats agent memory as a structured, first-class substrate for reasoning, centered around its four logical networks and three core operations.

### Memory Networks
1.  **World Facts**: Stores objective information about the external world.
2.  **Agent Experiences**: Records the agent's specific interactions and actions.
3.  **Synthesized Entity Summaries**: Compiles and refines information about entities.
4.  **Evolving Beliefs**: Represents the agent's dynamic understanding and hypotheses about the world.

### Core Operations
1.  **Retain**:
    *   Incrementally transforms conversational streams into a structured, queryable memory bank.
    *   Utilizes a temporal, entity-aware memory layer to organize new information.
2.  **Recall**:
    *   Efficiently retrieves relevant information from the memory bank based on query needs.
    *   Supports accessing information across different memory networks.
3.  **Reflect**:
    *   Reasons over the memory bank to produce answers or update existing beliefs.
    *   Ensures the information update process is traceable, enhancing the agent's explainability.

## Comparison with Previous Work

- **vs. Existing External Memory Systems**: Current systems treat memory as an external layer that extracts and retrieves snippets, whereas Hindsight views memory as a structured "first-class substrate" more deeply integrated into the reasoning process.
- **vs. Blurring Evidence and Inference**: Existing systems often blur the line between evidence and inference; Hindsight provides clearer distinctions by separating world facts, agent experiences, and evolving beliefs.
- **vs. Long-Horizon Information Organization**: Prior systems struggle to organize information effectively over long horizons; Hindsight's four-network structure and core operations support more efficient long-term information management.
- **vs. GPT-4o**: Hindsight, when paired with an open-source 20B model, outperforms full-context GPT-4o on long-horizon conversational memory benchmarks.
