## Motivation

长程AI智能体推理面临核心挑战：如何有效压缩不断增长的交互历史，使其适应有限的上下文窗口。
现有记忆系统通常将历史序列化为纯文本，导致token成本与长度线性增长，且常常将宝贵的预算浪费在低价值的细节上。
这限制了智能体在需要处理大量历史信息时的推理能力和效率。

## Contribution

- **多模态记忆智能体MemOCR**: 引入一种新的记忆系统，通过视觉布局自适应分配信息密度，从而在紧凑的上下文预算下改进长程推理。
- **结构化富文本记忆**: MemOCR维护一种结构化的富文本记忆（例如，包含标题、高亮），能够更好地组织和表达信息。
- **图像渲染与视觉优先级**: 将富文本记忆渲染成图像，智能体通过查阅该图像进行记忆访问，视觉上优先显示关键证据，同时积极压缩辅助细节。
- **预算感知强化学习训练**: 通过强化学习在预算感知目标下训练MemOCR，使其适应不同的压缩水平，确保在各种记忆预算下的鲁棒性。

## Method

MemOCR的核心思想是将结构化的富文本记忆渲染成图像，并通过视觉布局来管理信息密度。

1.  **构建结构化富文本记忆 (Structured Rich-Text Memory)**:
    *   智能体维护一个内部的富文本记忆表示，其中包含标题、高亮等结构化元素。
    *   这种表示允许更灵活地组织和突出重要信息，而不是简单的纯文本序列。

2.  **记忆渲染为图像 (Memory Rendering to Image)**:
    *   将结构化的富文本记忆渲染成一个图像。
    *   这个渲染过程是关键，它允许通过视觉布局（例如，字体大小、颜色、位置）来强调关键信息，并压缩次要细节。

3.  **智能体记忆访问 (Agent Memory Access)**:
    *   智能体不直接读取文本token，而是“查阅”渲染出的记忆图像。
    *   通过视觉处理，智能体能够识别图像中视觉上突出的关键证据，并忽略或快速浏览压缩的辅助信息。

4.  **预算感知强化学习 (Budget-Aware Reinforcement Learning)**:
    *   MemOCR通过RL进行训练，目标是优化在不同上下文预算下的性能。
    *   训练过程中，智能体会暴露于各种压缩水平，学习如何在有限的记忆空间内最大化信息效用。

## Motivation

Long-horizon agentic reasoning faces a core challenge: effectively compressing growing interaction histories to fit within a limited context window.
Most existing memory systems serialize history as plain text, leading to token costs that scale linearly with length and often wasting scarce budget on low-value details.
This limitation hinders agents' reasoning capabilities and efficiency when extensive historical information needs to be processed.

## Contribution

- **Multimodal Memory Agent MemOCR**: Introduces a novel memory system that adaptively allocates information density through visual layout, thereby improving long-horizon reasoning under tight context budgets.
- **Structured Rich-Text Memory**: MemOCR maintains a structured rich-text memory (e.g., with headings, highlights) that allows for better organization and expression of information.
- **Image Rendering and Visual Prioritization**: Renders the rich-text memory into an image that the agent consults for memory access, visually prioritizing crucial evidence while aggressively compressing auxiliary details.
- **Budget-Aware Reinforcement Learning Training**: Trains MemOCR with reinforcement learning under budget-aware objectives, exposing the agent to diverse compression levels to ensure robustness across varying memory budgets.

## Method

MemOCR's core idea is to render structured rich-text memory into an image and manage information density through visual layout.

1.  **Construct Structured Rich-Text Memory**:
    *   The agent maintains an internal rich-text memory representation, incorporating structured elements like headings and highlights.
    *   This representation allows for more flexible organization and emphasis of important information compared to a simple plain text sequence.

2.  **Memory Rendering to Image**:
    *   The structured rich-text memory is rendered into an image.
    *   This rendering process is crucial, as it enables emphasizing key information and compressing secondary details through visual layout (e.g., font size, color, position).

3.  **Agent Memory Access**:
    *   Instead of directly reading text tokens, the agent "consults" the rendered memory image.
    *   Through visual processing, the agent can identify visually prominent key evidence in the image, while ignoring or quickly scanning compressed auxiliary information.

4.  **Budget-Aware Reinforcement Learning**:
    *   MemOCR is trained using RL, with the objective of optimizing performance under various context budgets.
    *   During training, the agent is exposed to diverse compression levels, learning how to maximize information utility within limited memory space.

## Comparison with Previous Work

- **vs. Existing Text-Based Memory Systems**: Most existing systems serialize history as plain text, incurring linear token costs and treating all tokens uniformly. MemOCR adaptively allocates information density via visual layout, utilizing context budgets more efficiently.
- **vs. Traditional RAG Methods**: Traditional RAG typically retrieves and injects raw text snippets. MemOCR, by rendering an image, allows the agent to access memory in a more "global" and "visually prioritized" manner, rather than just through text matching.
- **vs. Earlier Multimodal Memory**: Earlier multimodal memory might simply juxtapose text and images. MemOCR's innovation lies in transforming text into a visual layout to achieve information compression and prioritization.
