## Motivation

现有AI智能体在复杂、长期的交互中，常因记忆能力不足而表现不佳，难以维持连贯的角色行为和对环境的深刻理解。
传统的记忆机制往往侧重于事实性知识或短期上下文，缺乏对智能体自身经历和所处场景的结构化、剧集式记忆。
这导致智能体在需要回顾特定事件、人物互动或场景细节时，检索效率低下且容易遗忘关键信息。

## Contribution

*   **角色与场景的剧集式记忆 (Character-and-Scene Episodic Memory)**：提出一种新颖的记忆结构，将智能体的经历编码为与特定角色和场景关联的剧集，而非单一的事件流。
*   **多模态记忆编码与检索 (Multimodal Memory Encoding and Retrieval)**：支持对文本、视觉等多种模态信息的统一记忆与高效检索，以适应复杂环境。
*   **情境感知记忆更新机制 (Context-Aware Memory Update Mechanism)**：设计了根据当前情境动态更新和整合新旧记忆的机制，确保记忆的及时性和相关性。
*   **长期交互与连贯性增强 (Enhanced Long-term Interaction and Coherence)**：通过结构化的剧集记忆，显著提升智能体在长期对话、任务执行中的行为连贯性和对环境的理解深度。

## Method

### 1. 记忆编码 (Memory Encoding)

*   **事件感知模块 (Event Perception Module)**：智能体从感知输入（文本、视觉等）中识别关键事件，并提取核心实体、动作和属性。
*   **剧集构建模块 (Episode Construction Module)**：将识别出的事件与当前交互的角色和场景关联起来，形成结构化的剧集。每个剧集包含时间戳、参与角色、发生场景、事件描述及相关多模态线索。

### 2. 记忆存储与组织 (Memory Storage and Organization)

*   **角色记忆图谱 (Character Memory Graph)**：维护一个图谱，节点代表智能体交互过的角色，边代表角色间的关系和互动历史。剧集与相关角色节点链接。
*   **场景记忆图谱 (Scene Memory Graph)**：维护另一个图谱，节点代表智能体经历过的场景，边代表场景间的空间或时间关系。剧集与相关场景节点链接。
*   **剧集索引 (Episode Index)**：为每个剧集创建多维嵌入，并建立高效索引，支持基于语义、时间、角色或场景的快速检索。

### 3. 记忆检索与推理 (Memory Retrieval and Reasoning)

*   **情境感知检索 (Context-Aware Retrieval)**：根据当前任务、对话上下文和智能体目标，动态计算检索查询。
*   **多路径检索 (Multi-Path Retrieval)**：同时利用角色记忆图谱、场景记忆图谱和剧集索引进行检索，以召回最相关的剧集。例如，如果当前与特定角色对话，会优先检索该角色的相关剧集；如果身处特定场景，则优先检索该场景的剧集。
*   **记忆整合与推理 (Memory Integration and Reasoning)**：检索到的剧集经过RAG机制整合到智能体的当前上下文，并通过LLM进行推理，生成更连贯、情境感知的响应或行为。

### 4. 记忆更新与遗忘 (Memory Update and Forgetting)

*   **增量更新 (Incremental Update)**：新发生的事件和剧集被增量地添加到记忆图谱和索引中。
*   **冲突解决与整合 (Conflict Resolution and Integration)**：当新信息与旧记忆冲突时，采用策略进行解决或整合，以保持记忆的一致性。
*   **遗忘机制 (Forgetting Mechanism)**：引入基于重要性、时间衰减或访问频率的遗忘机制，以管理记忆容量和保持记忆的相关性。

## Motivation

Existing AI agents often underperform in complex, long-term interactions due to insufficient memory capabilities, struggling to maintain coherent character behavior and a deep understanding of their environment.
Traditional memory mechanisms typically focus on factual knowledge or short-term context, lacking a structured, episodic memory of the agent's own experiences and the scenes it encounters.
This leads to inefficient retrieval and easy forgetting of critical information when agents need to recall specific events, character interactions, or scene details.

## Contribution

*   **Character-and-Scene Episodic Memory**: Proposes a novel memory structure that encodes agent experiences as episodes linked to specific characters and scenes, rather than a monolithic event stream.
*   **Multimodal Memory Encoding and Retrieval**: Supports unified memory and efficient retrieval of information from various modalities, such as text and vision, to adapt to complex environments.
*   **Context-Aware Memory Update Mechanism**: Designs a mechanism to dynamically update and integrate new and old memories based on the current context, ensuring memory timeliness and relevance.
*   **Enhanced Long-term Interaction and Coherence**: Significantly improves an agent's behavioral coherence and depth of environmental understanding in long-term dialogues and task execution through structured episodic memory.

## Method

### 1. Memory Encoding

*   **Event Perception Module**: The agent identifies key events from perceptual inputs (text, vision, etc.) and extracts core entities, actions, and attributes.
*   **Episode Construction Module**: Links identified events with currently interacting characters and scenes to form structured episodes. Each episode includes a timestamp, participating characters, scene of occurrence, event description, and relevant multimodal cues.

### 2. Memory Storage and Organization

*   **Character Memory Graph**: Maintains a graph where nodes represent characters the agent has interacted with, and edges represent relationships and interaction history between characters. Episodes are linked to relevant character nodes.
*   **Scene Memory Graph**: Maintains another graph where nodes represent scenes the agent has experienced, and edges represent spatial or temporal relationships between scenes. Episodes are linked to relevant scene nodes.
*   **Episode Index**: Creates multi-dimensional embeddings for each episode and builds an efficient index to support rapid retrieval based on semantics, time, character, or scene.

### 3. Memory Retrieval and Reasoning

*   **Context-Aware Retrieval**: Dynamically computes retrieval queries based on the current task, dialogue context, and agent goals.
*   **Multi-Path Retrieval**: Simultaneously uses the character memory graph, scene memory graph, and episode index for retrieval to recall the most relevant episodes. For example, if currently conversing with a specific character, episodes related to that character are prioritized; if in a particular scene, episodes from that scene are prioritized.
*   **Memory Integration and Reasoning**: Retrieved episodes are integrated into the agent's current context via a RAG mechanism and reasoned upon by an LLM to generate more coherent, context-aware responses or behaviors.

### 4. Memory Update and Forgetting

*   **Incremental Update**: Newly occurring events and episodes are incrementally added to the memory graphs and index.
*   **Conflict Resolution and Integration**: When new information conflicts with old memories, strategies are employed to resolve or integrate them to maintain memory consistency.
*   **Forgetting Mechanism**: Introduces a forgetting mechanism based on importance, temporal decay, or access frequency to manage memory capacity and maintain memory relevance.

## Comparison with Previous Work

*   **vs. RAG (Retrieval-Augmented Generation)**: CAST goes beyond simple document retrieval by providing more fine-grained, context-aware memory retrieval through structured character and scene episodic memory, especially suitable for agent interactions requiring long-term coherence.
*   **vs. Traditional episodic memory (e.g., experience replay in RL)**: CAST's episodic memory not only stores raw observations and actions but further abstracts them into semantic episodes linked to specific characters and scenes, supporting higher-level reasoning and recall.
*   **vs. Knowledge Graphs**: CAST's memory graphs focus on the agent's own experiences and interactions rather than general factual knowledge. It uses episodes as core elements, dynamically associating them with characters and scenes, making it more suitable for simulating an agent's "personal history."
*   **vs. Time-series based memory (e.g., Transformer's KV Cache)**: CAST's memory is not a simple linear time series but, through graph structures and semantic associations, supports non-linear, context-based memory jumps and recalls.
