## Motivation

现有对带记忆智能体的评估通常将记忆和行动孤立地分开评估。一类基准通过测试对过去对话或文本的召回能力来评估记忆，但未能捕捉记忆如何指导未来的决策。

另一类基准则侧重于在单会话任务中行动的智能体，这些任务不需要长期记忆。然而，在现实环境中，记忆和行动是紧密耦合的。

智能体在与环境交互时获取记忆，随后依赖这些记忆来解决未来的任务。为了捕捉这种真实场景，我们引入了MemoryArena。

## Contribution

- **统一评估基准**: 引入MemoryArena，一个用于在多会话Memory-Agent-Environment循环中评估智能体记忆的统一平台。
- **人机协作任务**: 包含由人类精心设计的智能体任务，这些任务具有明确的相互依赖子任务。
- **记忆蒸馏与应用**: 智能体必须从早期的行动和反馈中学习，将经验提炼成记忆，并随后利用这些记忆指导后续行动以解决整体任务。
- **多领域支持**: 支持在网页导航、偏好约束规划、渐进式信息搜索和顺序形式推理等多个领域进行评估。
- **揭示评估差距**: 发现现有长上下文记忆基准（如LoCoMo）上表现近乎饱和的智能体，在我们的智能体设置中表现不佳，从而揭示了当前智能体记忆评估的不足。

## Method

### MemoryArena 任务设计
1.  **多会话与相互依赖**: MemoryArena中的任务被设计为多会话且子任务之间相互依赖。这意味着智能体在早期会话中获得的记忆必须用于指导后续会话中的决策。
2.  **记忆获取与利用**: 智能体通过与环境交互来获取经验和反馈，并将这些经验蒸馏成可用于未来任务的记忆。
3.  **任务类型**:
    *   **网页导航 (Web Navigation)**: 智能体需要在多个网页之间导航以完成复杂任务。
    *   **偏好约束规划 (Preference-Constrained Planning)**: 智能体需要根据用户偏好进行规划和决策。
    *   **渐进式信息搜索 (Progressive Information Search)**: 智能体需要逐步搜索和整合信息以解决问题。
    *   **顺序形式推理 (Sequential Formal Reasoning)**: 智能体需要进行多步骤的逻辑推理来解决形式化问题。

### 评估指标
1.  **成功率 (Success Rate, SR)**: 衡量智能体成功完成任务的比例。
2.  **部分成功率 (Partial Success Rate, PS)**: 衡量智能体在多步骤任务中完成部分子任务的比例。
3.  **记忆利用效率**: 评估智能体如何有效地利用其记忆来改进决策和行动。

## Motivation

Existing evaluations of agents with memory typically assess memorization and action in isolation. One class of benchmarks evaluates memorization by testing recall of past conversations or text but fails to capture how memory is used to guide future decisions.

Another class focuses on agents acting in single-session tasks without the need for long-term memory. However, in realistic settings, memorization and action are tightly coupled.

Agents acquire memory while interacting with the environment, and subsequently rely on that memory to solve future tasks. To capture this setting, we introduce MemoryArena.

## Contribution

- **Unified Evaluation Gym**: Introduction of MemoryArena, a unified platform for benchmarking agent memory in multi-session Memory-Agent-Environment loops.
- **Human-Crafted Agentic Tasks**: Comprises human-crafted agentic tasks with explicitly interdependent subtasks.
- **Memory Distillation and Utilization**: Agents must learn from earlier actions and feedback by distilling experiences into memory, and subsequently use that memory to guide later actions to solve the overall task.
- **Multi-Domain Support**: Supports evaluation across web navigation, preference-constrained planning, progressive information search, and sequential formal reasoning.
- **Exposure of Evaluation Gap**: Reveals that agents with near-saturated performance on existing long-context memory benchmarks like LoCoMo perform poorly in our agentic setting, exposing a gap in current evaluations for agents with memory.

## Method

### MemoryArena Task Design
1.  **Multi-Session and Interdependent**: Tasks in MemoryArena are designed to be multi-session with interdependent subtasks. This means memory acquired in earlier sessions must guide decisions in subsequent sessions.
2.  **Memory Acquisition and Utilization**: Agents acquire experiences and feedback through interaction with the environment, distilling these experiences into memory for future tasks.
3.  **Task Types**:
    *   **Web Navigation**: Agents need to navigate across multiple web pages to complete complex tasks.
    *   **Preference-Constrained Planning**: Agents must plan and make decisions based on user preferences.
    *   **Progressive Information Search**: Agents need to progressively search and integrate information to solve problems.
    *   **Sequential Formal Reasoning**: Agents are required to perform multi-step logical reasoning to solve formal problems.

### Evaluation Metrics
1.  **Success Rate (SR)**: Measures the proportion of tasks successfully completed by the agent.
2.  **Partial Success Rate (PS)**: Measures the proportion of subtasks completed by the agent in multi-step tasks.
3.  **Memory Utilization Efficiency**: Assesses how effectively agents leverage their memory to improve decision-making and actions.

## Comparison with Previous Work

- **vs. LOCOMO (Maharana et al., 2024), LongMemEval (Wu et al., 2025), MemoryAgentBench (Hu et al., 2025a), MemoryBench (Ai et al., 2025)**: These benchmarks primarily evaluate memory recall but fail to capture how memory guides future decisions and actions in multi-session agentic tasks. They typically do not involve agent-environment interaction.
- **vs. WebArena (Zhou et al., ), WebShop (Yao et al., 2022), VeriGUI (Liu et al., 2025)**: These benchmarks focus on agentic actions in single-session tasks but generally do not require long-term memory. They lack the multi-session and interdependent subtask characteristics.
- **vs. Evo-Memory (Wei et al., 2025b), AgencyBench (Li et al., 2026a)**: While these works may incorporate memory and action to some extent, MemoryArena places a stronger emphasis on task interdependence, where agents must learn from earlier sessions and use their memory to guide subsequent decisions. MemoryArena provides a more explicit Memory-Agent-Environment loop.
