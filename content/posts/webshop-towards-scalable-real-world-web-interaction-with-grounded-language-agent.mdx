## Motivation

* 现有语言接地（language grounding）基准要么缺乏真实世界的语言元素，要么由于数据收集或反馈信号需要大量人工参与而难以扩展。
* 这导致难以开发和评估能够理解复杂指令并在真实网页环境中执行多样化操作的AI代理。
* WebShop旨在通过提供一个大规模、真实的模拟电商环境来弥补这一差距，以推动语言理解和决策制定能力更强的代理的发展。

## Contribution

* **WebShop环境**: 开发了一个包含118万真实产品和12,087条众包文本指令的模拟电商网站环境。
* **语言接地挑战**: 提出了理解组合指令、查询（重新）制定、理解和处理网页中嘈杂文本以及执行策略性探索等挑战。
* **人类演示数据集**: 收集了超过1,600个人类演示，用于训练和评估各种代理。
* **模型评估与分析**: 训练并评估了使用RL、IL以及预训练图像和语言模型的多种代理，并分析了代理和人类轨迹。
* **Sim-to-real迁移**: 证明了在WebShop上训练的代理在amazon.com和ebay.com上表现出非平凡的sim-to-real迁移能力。

## Method

1.  **环境构建**:
    *   WebShop是一个基于Python的模拟电商网站，使用Selenium WebDriver进行交互。
    *   它包含118万真实产品，这些产品从多个电商网站抓取，并存储在本地数据库中。
    *   网页渲染使用Jinja2模板引擎，确保了真实感和多样性。
2.  **指令收集**:
    *   通过众包平台收集了12,087条文本指令，每条指令描述了一个产品需求。
    *   指令设计为具有组合性，要求代理理解多个属性和偏好。
3.  **代理训练**:
    *   **模仿学习 (Imitation Learning, IL)**: 代理通过模仿人类专家演示来学习行为。
        *   使用BERT编码指令和网页文本，然后通过交叉注意力机制进行交互。
        *   训练目标是最小化代理动作与人类动作之间的交叉熵损失。
    *   **强化学习 (Reinforcement Learning, RL)**: 在IL模型基础上，使用策略梯度（Policy Gradient）方法进行微调。
        *   奖励函数考虑了产品属性匹配、选项选择和价格匹配等因素。
    *   **规则基线 (Rule-based Heuristics)**: 作为性能比较的基线，执行简单的搜索和选择策略。
4.  **评估指标**:
    *   任务成功率：代理能否成功找到、定制并购买符合指令要求的产品。
    *   详细评分：根据属性匹配、选项选择、产品类型和价格匹配等细分维度进行评分。

## Motivation

*   Existing benchmarks for grounding language in interactive environments either lack real-world linguistic elements or are difficult to scale due to substantial human involvement in data collection or feedback.
*   This makes it challenging to develop and evaluate AI agents capable of understanding complex instructions and performing diverse actions in real web environments.
*   WebShop aims to bridge this gap by providing a large-scale, realistic simulated e-commerce environment to foster the development of agents with stronger language understanding and decision-making abilities.

## Contribution

*   **WebShop Environment**: Developed a simulated e-commerce website environment with 1.18 million real-world products and 12,087 crowd-sourced text instructions.
*   **Language Grounding Challenges**: Identified challenges including understanding compositional instructions, query (re-)formulation, comprehending and acting on noisy text in webpages, and performing strategic exploration.
*   **Human Demonstrations Dataset**: Collected over 1,600 human demonstrations for training and evaluating a diverse range of agents.
*   **Model Evaluation and Analysis**: Trained and evaluated various agents using RL, IL, and pre-trained image and language models, and analyzed agent and human trajectories.
*   **Sim-to-Real Transfer**: Demonstrated that agents trained on WebShop exhibit non-trivial sim-to-real transfer when evaluated on amazon.com and ebay.com.

## Method

1.  **Environment Construction**:
    *   WebShop is a Python-based simulated e-commerce website, interacting via Selenium WebDriver.
    *   It features 1.18 million real-world products scraped from various e-commerce sites and stored in a local database.
    *   Webpages are rendered using the Jinja2 templating engine, ensuring realism and diversity.
2.  **Instruction Collection**:
    *   12,087 text instructions, each specifying a product requirement, were collected via crowdsourcing.
    *   Instructions are designed to be compositional, requiring agents to understand multiple attributes and preferences.
3.  **Agent Training**:
    *   **Imitation Learning (IL)**: Agents learn behaviors by imitating human expert demonstrations.
        *   BERT is used to encode instructions and webpage text, with cross-attention mechanisms for interaction.
        *   Training aims to minimize the cross-entropy loss between agent actions and human actions.
    *   **Reinforcement Learning (RL)**: Policy Gradient methods are used to fine-tune the IL model.
        *   The reward function considers factors like product attribute matching, option selection, and price matching.
    *   **Rule-based Heuristics**: Served as a baseline for performance comparison, executing simple search and selection strategies.
4.  **Evaluation Metrics**:
    *   Task success rate: Whether the agent successfully finds, customizes, and purchases a product meeting the instruction's requirements.
    *   Detailed scoring: Scores are broken down by attribute matching, option selection, product type, and price matching.

## Comparison with Previous Work

*   **vs. Existing Language Grounding Benchmarks**:
    *   WebShop offers richer real-world linguistic elements and product diversity, whereas many existing benchmarks are either oversimplified or rely on synthetic data.
    *   WebShop addresses scalability issues in existing benchmarks, which often require extensive human involvement, through automated environment interaction and reward computation.
*   **vs. Text-based Game Environments**:
    *   WebShop's web environment includes visual and layout information, not just pure text descriptions, making it closer to real-world web interaction.
    *   WebShop's instructions and products are real, not fictional, increasing the complexity and practical relevance of language understanding.
*   **vs. Existing Web Automation Tools**:
    *   WebShop focuses on language understanding and decision-making to fulfill complex user instructions, rather than just executing predefined scripts or simple web scraping tasks.
    *   The challenges in WebShop involve handling noisy webpage text and performing strategic exploration, which goes beyond the capabilities of traditional web automation tools.
