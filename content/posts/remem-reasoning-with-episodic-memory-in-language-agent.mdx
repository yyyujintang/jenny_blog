## Motivation

大型语言模型（LLM）在处理复杂、多跳或需要长期记忆的任务时，往往面临挑战。传统的RAG方法虽然能提供外部知识，但对于需要理解上下文、时间顺序和实体间复杂关系的情景记忆（episodic memory）任务效果不佳。

现有方法在构建和利用结构化记忆方面存在局限性，例如难以有效整合非结构化文本信息与结构化知识，或者在处理时间敏感信息时表现不足。这限制了LLM在需要深入理解历史交互和事件序列的应用中的能力。

## Contribution

- **情景记忆图谱（Episodic Memory Graph）**: 引入一种新的记忆结构，将非结构化文本信息转化为包含实体、事件和时间关系的结构化图谱，以捕捉更丰富的情景上下文。
- **多模态检索与推理机制**: 提出结合语义检索（semantic retrieval）和词法检索（lexical retrieval）的工具集，并利用图遍历（graph traversal）工具进行多跳推理，从而更有效地从情景记忆中提取相关信息。
- **拒绝回答能力（Refusal Capability）**: 增强模型识别无法回答问题并拒绝生成答案的能力，提高在信息不足情况下的可靠性。
- **在复杂任务上的SOTA性能**: 在情景回忆（episodic recollection）和情景推理（episodic reasoning）任务上，显著超越现有基线模型，证明了REMEm在处理复杂记忆相关任务上的优越性。

## Method

REMEm的核心在于其创新的情景记忆管理和检索机制：

1.  **构建情景记忆图谱**:
    *   **信息提取**: 从原始文本（如对话历史、文档）中识别关键实体、事件及其时间戳。
    *   **图谱构建**: 将提取出的信息组织成一个知识图谱，其中节点代表实体或事件，边表示它们之间的关系（例如，时间顺序、参与者、属性）。特别关注“要点（gists）”和“事实（facts）”的表示。
2.  **记忆检索工具集**:
    *   **语义检索（semantic_retrieve）**: 使用嵌入模型进行相似性搜索，召回与查询语义相关的记忆片段。
    *   **词法检索（lexical_retrieve）**: 基于关键词匹配进行检索，补充语义检索可能遗漏的精确匹配信息。
    *   **图谱探索工具（find_gist_contexts, find_entity_contexts）**: 允许LLM通过遍历图谱来查找与特定要点或实体相关的上下文信息，支持多跳推理。
3.  **LLM驱动的推理与回答**:
    *   LLM作为控制器，根据用户查询选择并调用上述检索和图谱探索工具。
    *   工具返回的结果被整合并作为LLM生成最终答案的上下文。
    *   模型还具备评估是否能回答问题的能力，对于无法回答的问题会拒绝生成答案。

## Comparison with Previous Work

- **vs. 传统RAG方法**: 传统的RAG主要依赖向量相似性检索，难以捕捉实体间的复杂关系和时间顺序。REMEm通过构建情景记忆图谱和图遍历工具，能进行更深层次的结构化推理。
- **vs. 基于大型嵌入模型的方法（如Qwen3-Embed, NV-Embed）**: 这些方法虽然在语义匹配上表现良好，但缺乏对记忆结构和事件序列的显式建模，在需要多跳推理和情景理解的任务中受限。
- **vs. 其他结构化记忆方法（如Mem0, Graphiti, HippoRAG 2）**: REMEm在图谱构建的粒度、工具集的丰富性以及对情景上下文的捕捉能力上有所增强。例如，Graphiti侧重于事实性知识，而REMEm更强调情景和时间信息。HippoRAG 2虽然也利用图结构，但REMEm通过更精细的要点/事实区分和工具调用策略，实现了更优的性能。

## Motivation

Large Language Models (LLMs) often struggle with complex, multi-hop reasoning tasks or those requiring long-term memory. While traditional RAG methods provide external knowledge, they are less effective for episodic memory tasks that demand an understanding of context, temporal order, and intricate relationships between entities.

Existing approaches for structured memory construction and utilization have limitations, such as difficulty in effectively integrating unstructured text with structured knowledge, or insufficient performance when dealing with time-sensitive information. This restricts LLMs' capabilities in applications requiring a deep understanding of historical interactions and event sequences.

## Contribution

- **Episodic Memory Graph**: Introduces a novel memory structure that transforms unstructured textual information into a structured graph containing entities, events, and temporal relationships, capturing richer episodic context.
- **Multi-modal Retrieval and Reasoning Mechanism**: Proposes a toolset combining semantic and lexical retrieval, along with graph traversal tools for multi-hop reasoning, to efficiently extract relevant information from episodic memory.
- **Refusal Capability**: Enhances the model's ability to identify unanswerable questions and refuse to generate responses, improving reliability in situations with insufficient information.
- **SOTA Performance on Complex Tasks**: Achieves significantly superior performance over existing baselines on episodic recollection and episodic reasoning tasks, demonstrating REMem's effectiveness in handling complex memory-related challenges.

## Method

REMem's core lies in its innovative episodic memory management and retrieval mechanism:

1.  **Episodic Memory Graph Construction**:
    *   **Information Extraction**: Identifies key entities, events, and their timestamps from raw text (e.g., dialogue history, documents).
    *   **Graph Building**: Organizes the extracted information into a knowledge graph where nodes represent entities or events, and edges denote their relationships (e.g., temporal order, participants, attributes). Special attention is given to representing "gists" and "facts."
2.  **Memory Retrieval Toolset**:
    *   **semantic_retrieve**: Uses embedding models for similarity search to recall memory fragments semantically related to the query.
    *   **lexical_retrieve**: Performs retrieval based on keyword matching, complementing semantic retrieval for precise matches.
    *   **Graph Exploration Tools (find_gist_contexts, find_entity_contexts)**: Allows the LLM to traverse the graph to find contextual information related to specific gists or entities, supporting multi-hop reasoning.
3.  **LLM-driven Reasoning and Answering**:
    *   The LLM acts as a controller, selecting and invoking the aforementioned retrieval and graph exploration tools based on the user's query.
    *   The results returned by the tools are integrated and provided as context for the LLM to generate the final answer.
    *   The model also possesses the ability to assess if a question can be answered, refusing to generate a response for unanswerable queries.

## Comparison with Previous Work

-   **vs. Traditional RAG Methods**: Traditional RAG primarily relies on vector similarity search, struggling to capture complex relationships and temporal order between entities. REMem, through its episodic memory graph and graph traversal tools, enables deeper, structured reasoning.
-   **vs. Large Embedding Model-based Approaches (e.g., Qwen3-Embed, NV-Embed)**: While these methods excel at semantic matching, they lack explicit modeling of memory structure and event sequences, limiting their effectiveness in tasks requiring multi-hop reasoning and episodic understanding.
-   **vs. Other Structured Memory Methods (e.g., Mem0, Graphiti, HippoRAG 2)**: REMem offers enhanced granularity in graph construction, a richer toolset, and improved capability in capturing episodic context. For instance, Graphiti focuses on factual knowledge, whereas REMem emphasizes episodic and temporal information. While HippoRAG 2 also leverages graph structures, REMem achieves superior performance through a more refined distinction between gists/facts and a sophisticated tool-calling strategy.
