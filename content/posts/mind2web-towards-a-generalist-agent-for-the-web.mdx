## Motivation

现有的网络代理数据集通常使用模拟网站或仅覆盖有限的网站和任务，这限制了通用型网络代理的开发和评估。
真实世界的网站具有高度多样性和复杂性，需要代理能够处理各种用户交互模式和动态内容。
因此，迫切需要一个包含真实世界网站、多样化任务和广泛用户交互模式的数据集，以推动通用型网络代理的研究。

## Contribution

- **Mind2Web数据集**: 首个用于开发和评估通用型网络代理的数据集，包含来自137个网站、31个领域的2,000多个开放式任务。
- **真实世界网站**: 使用真实世界的网站而非模拟或简化网站，更贴近实际应用场景。
- **多样化的用户交互模式**: 涵盖广泛的用户交互模式，为代理学习复杂任务提供丰富数据。
- **LLM探索**: 基于Mind2Web对使用大型语言模型（LLMs）构建通用型网络代理进行了初步探索。
- **HTML过滤方法**: 提出通过小型LM过滤原始HTML的方法，显著提高LLMs的效率和有效性。

## Method

### Mind2Web数据集构建
1. **网站选择**: 从Alexa Top 5000中选择多样化的网站，涵盖不同领域。
2. **任务收集**: 众包收集开放式任务，确保任务的复杂性和多样性。
3. **动作序列标注**: 众包标注每个任务的详细动作序列，包括元素选择和操作类型。

### 基于LLM的通用型网络代理探索
1. **HTML预处理**: 使用小型LM过滤原始HTML，提取与任务相关的关键信息，减少LLM的输入长度。
2. **LLM决策**: 将过滤后的HTML和任务指令输入LLM，由LLM生成下一步动作。
3. **动作执行**: 根据LLM的输出执行相应的网页操作。

## Comparison with Previous Work

- **vs. MiniWoB++ / WebShop**: 这些数据集使用模拟或简化的网站，而Mind2Web使用真实世界的网站，更具挑战性和实用性。
- **vs. RUSS / PixelHelp / META-GUI / MoTIF**: 这些数据集虽然包含真实世界的网站或移动应用，但其网站和任务的多样性远不及Mind2Web，无法支持通用型代理的开发。
- **vs. 现有LLM方法**: 现有LLM在处理原始、大型的真实世界HTML时效率和效果受限，本文提出的HTML过滤方法有效解决了这一问题。

## Motivation

Existing datasets for web agents often rely on simulated websites or cover only a limited range of websites and tasks, which restricts the development and evaluation of generalist web agents.
Real-world websites exhibit high diversity and complexity, requiring agents to handle various user interaction patterns and dynamic content.
Therefore, there is a pressing need for a dataset that features real-world websites, diverse tasks, and a broad spectrum of user interaction patterns to advance research in generalist web agents.

## Contribution

- **Mind2Web Dataset**: The first dataset for developing and evaluating generalist web agents, comprising over 2,000 open-ended tasks collected from 137 websites across 31 domains.
- **Real-World Websites**: Utilizes real-world websites instead of simulated or simplified ones, making it more relevant to practical applications.
- **Diverse User Interaction Patterns**: Covers a broad spectrum of user interaction patterns, providing rich data for agents to learn complex tasks.
- **LLM Exploration**: Conducts an initial exploration of using Large Language Models (LLMs) for building generalist web agents based on Mind2Web.
- **HTML Filtering Method**: Proposes a method to filter raw HTML with a small LM, significantly improving the effectiveness and efficiency of LLMs.

## Method

### Mind2Web Dataset Construction
1. **Website Selection**: Diverse websites are chosen from the Alexa Top 5000, covering various domains.
2. **Task Collection**: Open-ended tasks are crowdsourced to ensure complexity and variety.
3. **Action Sequence Annotation**: Detailed action sequences for each task, including element selection and operation types, are crowdsourced.

### LLM-based Generalist Web Agent Exploration
1. **HTML Preprocessing**: A small LM is used to filter raw HTML, extracting key task-relevant information and reducing the input length for the LLM.
2. **LLM Decision-Making**: The filtered HTML and task instructions are fed into the LLM, which then generates the next action.
3. **Action Execution**: The corresponding web operation is executed based on the LLM's output.

## Comparison with Previous Work

- **vs. MiniWoB++ / WebShop**: These datasets use simulated or simplified websites, whereas Mind2Web employs real-world websites, making it more challenging and practical.
- **vs. RUSS / PixelHelp / META-GUI / MoTIF**: While these datasets include real-world websites or mobile apps, their diversity in websites and tasks is significantly less than Mind2Web, thus insufficient for developing generalist agents.
- **vs. Existing LLM Approaches**: Current LLM methods struggle with the efficiency and effectiveness of processing raw, large real-world HTML; the proposed HTML filtering method effectively addresses this limitation.
